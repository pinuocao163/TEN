# 摘要

多模态会话情感识别需要在对话上下文中联合理解文本语义、语音韵律和视觉表情。现有 Transformer、图神经网络和提示学习方法能够提升跨模态建模能力，但多数方法仍将融合过程压缩为隐式高维表示：模型给出情感类别，却难以说明它参考了哪些相似案例、为何信任某一模态，以及外部知识在多大程度上改变了原始多模态判断。针对这一问题，本文提出一种证据约束的多模态认知残差融合框架，在 TEN 多模态 Transformer 骨干之上构建“检索证据—认知解析—可靠性分配—残差校正”的可追溯决策链。具体而言，本文首先从训练集中检索与当前话语在文本上下文和音视频状态上相似的样本，以数据集内部证据约束大语言模型推理；随后将大语言模型输出解析为结构化认知变量，包括情感软分布、VAD 评价、模态提示、推理置信度和检索质量；进一步地，模型基于这些认知变量动态估计文本、语音、视觉和认知推理四类信息源的可靠性，并通过质量感知残差门控控制认知信息对 TEN 表示的修正幅度。与此同时，本文利用 VAD 空间构造情感语义约束，使表示学习兼顾离散类别判别和连续情感结构。实验在 IEMOCAP 和 MELD 数据集上进行，结果表明，本文方法能够在保持有竞争力识别性能的同时，提供更清晰的证据来源、模态贡献和推理注入强度，从而提升多模态会话情感识别的机制可解释性与可信度。

# 引言

多模态会话情感识别旨在根据对话中的文本内容、语音韵律、视觉表情和上下文交互关系，识别每个话语所表达的情感状态。该任务是情感计算、自然语言处理和可信人机交互中的重要问题，在智能客服、社交媒体理解、心理健康分析和具身智能交互等场景中具有广泛应用价值。与单句情感分类不同，会话情感识别通常面临三类困难：第一，情感表达具有强上下文依赖，同一句话在不同对话历史和说话人状态下可能呈现不同情绪；第二，文本、语音和视觉并非始终一致，讽刺、沉默、遮挡、噪声和个体表达差异都会造成模态冲突；第三，情感类别之间存在连续语义关系，例如 happy 与 excited、angry 与 frustrated、neutral 与 sad 往往只在唤醒度、控制感或情绪强度上存在细粒度差别。已有研究从说话人关系、情绪转移、图结构上下文和多模态交互等角度提升会话情感识别能力 [1][22][34][53]，但复杂场景下的“为什么这样预测”仍然难以回答。

近年来，基于 Transformer、图神经网络和多模态提示学习的方法显著推动了 MERC 发展。MPT 将多模态提示和混合对比学习引入 Transformer 框架 [70]，DQ-Former 通过动态模态优先级刻画样本级模态贡献差异 [76]，VEGA 利用视觉原型强化情感语义对齐 [47]，超图、关系图和跨模态语义图方法也进一步增强了会话结构建模能力 [39][43][45][51]。然而，这些方法大多仍依赖端到端学习得到的隐式融合权重。即使模型内部存在注意力或门控机制，其权重也往往缺少可验证的外部证据支撑。当模型面对低质量视觉、语音噪声或文本歧义时，研究者很难判断模型究竟是依据相似情感案例、某一模态线索，还是仅仅依赖数据偏置完成预测。这种机制不透明性限制了模型在高可信应用中的使用。

大语言模型展现出的上下文理解、情感解释和常识推理能力为解决上述问题提供了新机会。近期工作已经尝试利用 LLM 建模说话人特征、语音细节、心理状态和对话场景信息，以增强 ERC/MERC 中的高层语义理解能力 [16][31][41][46]。知识增强方法也表明，情绪常识、人格信息、情绪原因和情绪转移知识能够为复杂会话中的情感判断提供重要补充 [6][7][20][58]。然而，直接把 LLM 输出作为情感预测或后验解释并不充分。一方面，LLM 可能过度依赖通用知识，与特定数据集的标签边界和标注风格不一致；另一方面，后验生成的解释并不一定真实参与神经模型决策，仍无法解释模型内部融合过程；此外，当输入证据不足或检索样本质量较低时，LLM 推理可能引入新的噪声。因此，关键问题不是简单“接入 LLM”，而是如何让 LLM 推理受到任务数据约束，并以可度量、可筛选、可控的方式参与多模态决策。

本文围绕这一问题提出证据约束的多模态认知残差融合框架。核心思想是将 LLM 从外部预测器转化为可解释的认知信号提供者：模型首先通过多模态 RAG 从训练集中检索相似样本，使 LLM 推理对齐数据集内部标签边界；然后将 LLM 输出解析为情感软分布、VAD 评价、模态提示、置信度和检索质量；最后将这些认知变量注入 TEN 骨干的可靠性融合模块，由模型动态决定文本、语音、视觉和 LLM 认知分支的贡献，并通过残差强度控制认知推理对原始多模态表示的修正程度。这样，最终预测不再只是一个黑箱分类结果，而是可以沿着检索样本、认知变量、可靠性权重和残差门控逐层追溯。

本文的主要贡献如下。

1. 提出一种证据约束的多模态 RAG-LLM 认知推理机制。该机制联合文本上下文相似度和音视频统计相似度检索训练集样本，使 LLM 推理受到数据集内部案例约束，从而降低其偏离任务标签体系的风险。
2. 构建结构化认知变量，将 LLM 输出从自然语言解释转化为可计算信号。认知变量包含情感软分布、VAD appraisal、模态提示、置信度、检索质量和 VAD 指导向量，既能参与模型训练，也能支撑解释分析。
3. 设计可靠性感知的认知残差融合模块。模型根据认知变量动态分配文本、语音、视觉和 LLM 认知分支的可靠性，并通过质量感知残差强度控制认知表示对 TEN 基础表示的插值式修正，避免低质量推理无条件覆盖多模态骨干判断。
4. 引入 VAD 引导的情感语义约束。本文将离散情感标签与 valence-arousal-dominance 连续空间结合，使模型在区分类别的同时保留情感之间的强度、极性和控制感结构，尤其适用于 happy/excited、angry/frustrated、neutral/sad 等易混淆类别。

# 相关工作

## 2.1 上下文与图结构会话情感识别

会话情感识别的核心在于建模上下文依赖、说话人交互和情绪状态变化。DualGATs 通过双图注意力网络分别建模 discourse-aware graph 和 speaker-aware graph [1]，Curriculum Learning Meets DAG 将有向无环图引入多模态情感识别并结合课程学习 [22]，Adaptive Graph Learning 从数据中自适应学习多模态会话图结构 [34]。近期方法进一步关注高阶关系和复杂交互，例如 Graph Spectrum 从图频谱角度分析 MERC [38]，Dynamic Interactive Bimodal Hypergraph Networks 使用超图建模双模态高阶关系 [39]，Hybrid Relational Graphs 和 MATCH 分别从情感语义对齐和模态校准超图角度增强多模态融合 [43][45]。这些方法说明结构化上下文对 ERC/MERC 至关重要，但多数方法仍主要在神经特征或图拓扑内部学习关系，缺少可追溯外部证据。

## 2.2 多模态融合与可靠性建模

文本、语音和视觉在不同话语中的可靠性并不相同。CMCF-SRNet 通过跨模态上下文融合和语义细化提升模态交互质量 [3]，MultiEMO 利用相关性感知注意力框架建模多模态关系 [4]，Joyful 将联合模态融合与图对比学习结合起来 [17]。近期研究进一步关注模态平衡、动态优先级和可信输出。MERC Calibration 关注多模态情感识别中的校准问题 [73]，Ada2I 缓解模态不平衡 [74]，DQ-Former 根据样本动态查询不同模态优先级 [76]，DRKF 通过表示解耦和知识融合增强多模态情感识别 [77]。与这些方法相比，本文不仅学习隐式模态权重，还将 LLM 生成的模态提示、置信度、VAD 评价和检索质量显式纳入可靠性分配，使融合权重具备可解释来源。

## 2.3 知识增强、提示学习与 LLM 情感推理

会话情感往往依赖隐含常识、情绪原因、说话人特征和情绪转移规律。ECoK 挖掘情绪常识知识图谱 [6]，EmoTransKG 利用情绪知识图谱揭示情绪转移 [7]，SKIER 将符号知识融入会话情感识别 [32]，MKE-IGN 通过多知识增强交互图网络融合外部知识和多模态上下文 [21]，UniMEEC 将情感识别与情感原因分析统一建模 [20]。随着 LLM 发展，提示学习和 LLM 适配方法也被引入 MERC。MSE-Adapter 设计轻量插件赋予 LLM 多模态情感分析能力 [41]，LaERC-S 利用 LLM 挖掘说话人特征 [31]，Beyond Silent Letters 强调语音细节对 LLM 情感理解的重要性 [16]，DialogueMMT 利用对话场景理解增强多模态多任务调优 [11]。这些方法证明 LLM 可以提供高层语义补充；本文进一步强调，LLM 推理需要被训练集证据约束，并以结构化认知变量的形式真实进入模型内部计算链条。

## 2.4 情感语义约束与可信 MERC

除准确率之外，MERC 还需要面对类别语义相近、模态缺失、样本不均衡和模型可信度不足等问题。Emotion-Anchored Contrastive Learning 使用情感锚点约束表示空间 [13]，Dual Contrastive Learning Framework 通过双重对比学习增强多模态表示 [30]，PEMV 利用近邻情绪均值向量改善情感空间分布 [15]。缺失模态和说话人泛化也是近期重要方向 [8][48][57][61][64]。本文与上述方向互补：通过 RAG 证据、认知变量、可靠性门控和残差强度，把性能提升与机制解释连接起来，使模型不仅能预测情感类别，还能回答“依据什么证据、信任哪些模态、LLM 推理产生了多大影响”。

# 方法

## 3.1 任务定义与总体框架

给定一个包含 $N$ 个话语的多模态对话：

$$
D=\{u_1,u_2,\ldots,u_N\},
$$

每个话语 $u_i$ 包含文本、语音和视觉特征：

$$
u_i=\{x_i^t,x_i^a,x_i^v\}.
$$

模型目标是在对话上下文中预测话语级情感标签：

$$
\hat{y}_i=f(x_i^t,x_i^a,x_i^v,D),\quad \hat{y}_i\in\{1,\ldots,C\}.
$$

本文的核心目标不是简单提高分类器容量，而是把多模态情感识别建模为一条可追溯、可控制的认知决策链。传统多模态融合通常直接学习：

$$
(x_i^t,x_i^a,x_i^v)\rightarrow h_i^{base}\rightarrow \hat{y}_i,
$$

其中 $h_i^{base}$ 是端到端训练得到的隐式表示。该范式虽然有效，但难以解释预测依据。本文在此基础上增加一条由训练集证据驱动的认知分支：

$$
\text{Multimodal Input}\rightarrow h_i^{base}
,\quad
\text{Retrieved Evidence}\rightarrow c_i
,\quad
(h_i^{base},c_i)\rightarrow h_i^{final}\rightarrow \hat{y}_i.
$$

具体而言，本文方法包含四个阶段。

1. 多模态交互编码。使用 TEN 骨干从文本、语音和视觉中学习基础多模态表示 $h_i^{base}$。
2. 多模态证据检索。从训练集中检索文本语义和音视频状态相似的标注样本，形成证据集合 $\mathcal{R}_i$。
3. 结构化认知解析。将当前话语和检索证据输入 LLM，得到情感软分布、VAD appraisal、模态提示、置信度和检索质量等认知变量 $c_i$。
4. 质量受控残差融合。根据 $c_i$ 动态估计文本、语音、视觉和 LLM 认知分支的可靠性，并以残差插值方式修正 $h_i^{base}$。

这一设计有两个关键区别。第一，LLM 不直接替代分类器，而是作为认知变量生成器，为神经模型提供可解释的中间信号。第二，LLM 认知信息不是后验解释，而是通过可靠性门控、残差控制和辅助训练目标真实参与预测过程。因此，模型不仅输出情感类别，还能解释“参考了哪些证据、信任哪些模态、LLM 认知信息修改了多少基础表示”。

## 3.2 TEN 多模态交互编码

项目中使用预处理好的文本、语音和视觉特征。文本特征维度为 1024，视觉 denseface 特征维度为 342；语音特征在 IEMOCAP 上为 1582 维，在 MELD 上为 300 维。由于不同模态维度不一致，模型首先通过一维卷积将三类模态映射到统一 1024 维隐空间：

$$
h_i^m=\phi_m(x_i^m),\quad m\in\{t,a,v\}.
$$

其中 $\phi_m(\cdot)$ 对应代码中的 `textf_input`、`acouf_input` 和 `visuf_input`。该投影步骤有两个作用：一是消除不同模态原始特征维度差异，二是把三类模态映射到同一 Transformer 隐空间。随后，模型加入位置编码和说话人嵌入：

$$
\tilde{h}_i^m=h_i^m+p_i+s_i.
$$

其中，$p_i$ 表示话语位置，$s_i$ 表示说话人身份。IEMOCAP 使用 2 个说话人嵌入，MELD 使用 9 个说话人嵌入。这样做能够让模型同时感知话语顺序和说话人状态变化。

TEN 骨干包含九条 Transformer 分支，对应三类目标模态与三类源模态之间的全部交互：

$$
z_i^{m\leftarrow n}=\operatorname{Transformer}_{m\leftarrow n}(\tilde{h}^m,\tilde{h}^n),
\quad m,n\in\{t,a,v\}.
$$

当 $m=n$ 时，Transformer 建模单模态内部上下文；当 $m\neq n$ 时，Transformer 以源模态 $n$ 为键值、目标模态 $m$ 为查询，建模跨模态信息补充。代码中九条分支分别为 `t_t`、`a_t`、`v_t`、`a_a`、`t_a`、`v_a`、`v_v`、`t_v` 和 `a_v`。对每个目标模态，模型将来自文本、语音和视觉源模态的交互结果拼接并降维：

$$
r_i^m=W_m[z_i^{m\leftarrow t};z_i^{m\leftarrow a};z_i^{m\leftarrow v}].
$$

这里的 $r_i^t$、$r_i^a$ 和 $r_i^v$ 可以理解为三种“被其他模态增强后的目标模态表示”。随后，TEN 使用多模态门控得到基础融合表示：

$$
h_i^{base}=\operatorname{Gate}(r_i^t,r_i^a,r_i^v).
$$

该门控由神经网络根据三类表示自动学习。它能够捕获细粒度多模态交互，但门控权重本身仍主要来自隐空间计算，缺少外部证据约束，也无法明确说明为什么某个样本更依赖文本、语音或视觉。因此，本文在 $h_i^{base}$ 之上引入认知证据链，而不是替换 TEN 骨干。

## 3.3 多模态 RAG 证据构建

为了让 LLM 推理贴近数据集标签体系，本文不直接让 LLM 根据当前话语作答，而是先从训练集中检索相似情感案例。这样做的动机是：MELD 和 IEMOCAP 的标签边界并不完全等同于通用语义空间中的情绪概念。例如 excited 与 happy、frustrated 与 angry 的区别往往依赖数据集标注习惯和具体多模态线索。训练集中的相似案例能够为 LLM 提供局部标签边界。

对目标话语 $u_i$，本文构造两个互补的相似度。文本相似度 $s_{text}$ 由当前话语上下文的 TF-IDF 表示计算得到；音视频状态相似度 $s_{av}$ 由语音和视觉统计向量计算得到。最终检索得分为：

$$
s(u_i,u_j)=\lambda s_{text}(u_i,u_j)+(1-\lambda)s_{av}(u_i,u_j).
$$

项目实现中，文本检索采用 1-2 gram TF-IDF，最大特征数为 50000。音视频统计向量由 8 个维度组成：

$$
e_i^{av}=[E_a,\sigma_a,P_a,\Delta_a,E_v,\sigma_v,P_v,\Delta_v],
$$

其中，$E$ 表示平均能量，$\sigma$ 表示变化程度，$P$ 表示峰值强度，$\Delta$ 表示相邻话语偏移；下标 $a$ 和 $v$ 分别表示语音和视觉。统计向量经过标准化后用于计算多模态状态相似度。模型选择得分最高的 $K$ 个训练样本作为证据集合：

$$
\mathcal{R}_i=\operatorname{TopK}_{u_j\in\mathcal{D}_{train}}s(u_i,u_j).
$$

检索证据以带标签案例形式写入 prompt。每个案例包含对话上下文、gold label 和音视频统计。LLM 被明确要求“利用检索案例校准标签边界，但不要机械复制邻居标签”。因此，RAG 分支的功能不是最近邻分类，而是把数据集内部的局部决策边界显式暴露给 LLM，使其生成的认知变量更符合当前任务。

本文还为检索证据计算质量分数 $\rho_i$。代码中 $\rho_i$ 综合考虑检索 top score、平均得分、文本相似度、音视频相似度以及检索样本与 LLM 预测标签的一致性。高 $\rho_i$ 表示检索案例更集中、更可靠；低 $\rho_i$ 表示证据分散或与 LLM 判断不一致，后续残差融合会自动降低其影响。

## 3.4 结构化认知变量

基于当前话语、对话上下文和检索证据 $\mathcal{R}_i$，LLM 被要求输出一个 JSON 对象，而不是自由形式解释。输出字段包括 `label`、`confidence`、`rationale`、`appraisal` 和 `modality_hint`。这样设计的原因是，自然语言解释难以稳定参与神经网络计算，而结构化变量可以被解析、筛选、监督和融合。

解析后得到认知特征：

$$
c_i=[p_i^{llm};a_i;q_i;\gamma_i;\rho_i;g_i].
$$

其中各部分含义如下。

1. $p_i^{llm}\in\mathbb{R}^{C}$ 是 LLM 情感软分布。若 LLM 输出 hard label，则代码根据置信度构造平滑分布；若输出概率分布，则直接归一化使用。该分布不仅提供预测类别，也包含类别间相似性。
2. $a_i\in\mathbb{R}^{4}$ 是 VAD appraisal，包括 valence、arousal、dominance 和 certainty。它把离散情感映射到连续情感空间，帮助区分语义接近类别。
3. $q_i\in\mathbb{R}^{3}$ 是模态提示，表示 LLM 认为文本、语音和视觉在当前样本中的相对重要性。
4. $\gamma_i\in\mathbb{R}$ 是 LLM 置信度，用于估计推理本身是否可信。
5. $\rho_i\in\mathbb{R}$ 是 RAG 检索质量，用于估计证据集合是否可靠。
6. $g_i\in\mathbb{R}^{3}$ 是 VAD guidance，由 appraisal 派生得到，用于将连续情感结构进一步转化为门控可用的指导信号。

因此认知特征维度为：

$$
\dim(c_i)=C+12.
$$

VAD guidance 的三维构造为：

$$
g_i=[\max(v,0)\cdot a,\ \max(-v,0)\cdot\max(a,0)\cdot d,\ \max(-v,0)\cdot\max(-a,0)],
$$

其中 $v,a,d$ 分别表示 valence、arousal 和 dominance。三维含义分别对应正向激活、负向控制轴和负向低唤醒状态。该设计使模型能够把 happy/excited、angry/frustrated、sad/neutral 等差异显式编码到认知特征中。

如果 LLM 输出缺失或格式不稳定，代码会根据情感先验补全 appraisal 和 modality hint。例如 excited 更强调高 arousal，angry/frustrated 更强调语音线索，neutral 更强调文本稳定性。这样可以保证认知特征在实际生成噪声下仍保持数值稳定。

## 3.5 可靠性感知认知残差融合

认知特征首先被投影到与 TEN 表示一致的隐空间：

$$
h_i^c=\psi(c_i).
$$

这里的 $\psi(\cdot)$ 对应代码中的两层投影结构：线性变换、ReLU 和 LayerNorm。它把低维认知变量转化为可以与 $r_i^t,r_i^a,r_i^v$ 对齐的隐表示 $h_i^c$。

接下来，模型同时观察文本、语音、视觉和认知表示，计算四类信息源的基础可靠性 logits：

$$
o_i=W_r[r_i^t;r_i^a;r_i^v;h_i^c].
$$

仅依赖 $o_i$ 仍然是隐式门控。为使可靠性分配受到认知变量显式影响，本文加入认知偏置：

$$
\beta_i=\operatorname{softmax}(W_r[r_i^t;r_i^a;r_i^v;h_i^c]+\Delta(c_i)).
$$

其中，$\beta_i=[\beta_i^t,\beta_i^a,\beta_i^v,\beta_i^c]$，分别表示文本、语音、视觉和认知推理的可靠性。认知偏置 $\Delta(c_i)$ 由两部分组成。第一部分来自模态提示和推理可信度：

$$
h_i^{hint}=[q_i^t,q_i^a,q_i^v,\gamma_i(0.5+0.5\rho_i)].
$$

其中第四维是 reasoning trust，表示 LLM 分支可信程度。第二部分来自 VAD appraisal 和 VAD guidance：

$$
h_i^{vad}=[a_i;g_i].
$$

二者分别通过线性层映射到四维 gate bias，并加到基础 logits 上。这样，模型的可靠性权重不再只是黑箱学习结果，而是同时受到“LLM 认为哪个模态更重要”“当前情感在 VAD 空间中处于什么区域”“检索证据是否可靠”等因素调节。

若某个话语没有对应 cache，或 cache 未通过质量过滤，代码会对认知分支进行 mask，使 $\beta_i^c$ 对应 logit 变为极小值，从而避免缺失或低质量认知信息参与融合。可靠性感知表示为：

$$
h_i^{rel}=\beta_i^t r_i^t+\beta_i^a r_i^a+\beta_i^v r_i^v+\beta_i^c h_i^c.
$$

直接使用 $h_i^{rel}$ 替代 $h_i^{base}$ 会带来风险：如果 LLM 推理错误，模型可能被外部认知噪声误导。为此，本文设计插值式残差校正：

$$
h_i^{final}=h_i^{base}+\eta_i(h_i^{rel}-h_i^{base}).
$$

该公式与代码中的 `all_transformer_out + reasoning_alpha * (reasoning_out - all_transformer_out)` 完全一致。它可以理解为：$h_i^{base}$ 是 TEN 的原始判断，$h_i^{rel}-h_i^{base}$ 是认知分支建议的修正方向，$\eta_i$ 决定是否接受以及接受多少修正。残差强度由可学习全局初值、LLM 置信度和 RAG 质量共同决定：

$$
\eta_i=\sigma(\theta_\eta)\cdot(0.5+\sigma(W_\eta[\gamma_i;\rho_i]))\cdot \gamma_i(0.5+0.5\rho_i).
$$

该控制器具有三层保护。第一，$\sigma(\theta_\eta)$ 是全局可学习残差上限，避免模型一开始过度依赖 LLM。第二，$W_\eta[\gamma_i;\rho_i]$ 根据样本质量动态调节残差强度。第三，$\gamma_i(0.5+0.5\rho_i)$ 将低置信度或低检索质量样本的残差进一步压低。因此，当检索质量低或 LLM 置信度低时，模型主要保留 TEN 基础判断；当证据充分且推理可信时，认知表示才会以更大强度修正基础表示。

最终预测为：

$$
\hat{p}_i=\operatorname{softmax}(W_o h_i^{final}+b_o).
$$

这一模块是本文方法的关键创新之一：它不是把 LLM 输出拼接到特征上，也不是简单蒸馏 LLM 标签，而是让 LLM 认知变量同时参与可靠性分配和残差强度控制，从而实现“可解释但不过度依赖”的认知融合。

## 3.6 训练目标

模型总损失由主分类损失、单模态监督、自蒸馏、LLM 认知蒸馏、可靠性监督和 VAD 对比约束组成：

$$
\mathcal{L}=\mathcal{L}_{ce}+\lambda_{uni}\mathcal{L}_{uni}
+\lambda_{sd}\mathcal{L}_{sd}
+\lambda_{llm}\mathcal{L}_{llm}
+\lambda_{rel}\mathcal{L}_{rel}
+\lambda_{vad}\mathcal{L}_{vad}.
$$

其中，$\mathcal{L}_{ce}$ 为主分类交叉熵，监督最终融合分支预测真实情感标签；$\mathcal{L}_{uni}$ 分别监督文本、语音和视觉单模态输出，避免某些模态在训练中退化；$\mathcal{L}_{sd}$ 是 TEN 原有自蒸馏项，用最终融合分支的 softened distribution 约束单模态分支，使单模态表示与融合表示保持一致。本文新增的三个关键目标为 $\mathcal{L}_{llm}$、$\mathcal{L}_{rel}$ 和 $\mathcal{L}_{vad}$。

LLM 认知蒸馏使用 LLM 情感软分布约束最终预测：

$$
\mathcal{L}_{llm}=\frac{1}{|\Omega|}\sum_{i\in\Omega}w_i
\operatorname{KL}(p_i^{llm}\Vert \hat{p}_i).
$$

权重 $w_i$ 由 LLM mask、推理置信度和检索质量共同决定，只有满足质量阈值和置信度阈值的样本才参与有效蒸馏。这样，模型不会盲目模仿所有 LLM 输出，而是只从高可信认知判断中学习类别间软关系。

可靠性监督鼓励模型学习与认知提示一致的信息源权重：

$$
\mathcal{L}_{rel}=\frac{1}{|\Omega|}\sum_{i\in\Omega}\|\beta_i-\beta_i^\ast\|_2^2.
$$

其中，$\beta_i^\ast$ 由 LLM 模态提示、VAD 派生模态提示、置信度、检索质量和 VAD 强度共同构造。该目标使可靠性门控不只是自由学习，而是被结构化认知变量牵引。例如，当 LLM 判断当前话语的情绪主要由语音唤醒度体现，并且检索证据可靠时，目标分布会提高 audio 和 cognitive 分支权重；当 LLM 置信度较低时，认知分支目标权重会被压低。

VAD 引导对比损失用于约束表示空间。对于同标签样本，VAD 越相似，正样本吸引越强；对于易混淆负样本，模型施加额外间隔约束。IEMOCAP 中 happy/excited、angry/frustrated、neutral/sad 被显式作为易混淆对，因为它们往往只在 arousal、dominance 或 valence 强度上存在细微差别。该目标使最终表示不仅服务于离散分类，也保持连续情感空间中的语义结构。

整体而言，训练目标形成三层约束：交叉熵保证判别能力，LLM 蒸馏和可靠性监督保证认知变量真正参与学习，VAD 对比学习保证表示空间符合情感心理结构。这使模型的创新点不仅体现在结构上，也体现在训练信号的组织方式上。

## 3.7 可解释决策链

本文方法的解释性来自模型内部计算链条，而不是预测后的自然语言说明。对于任一话语 $u_i$，模型可以输出或统计以下证据链：

$$
\mathcal{R}_i\rightarrow c_i\rightarrow \beta_i\rightarrow \eta_i\rightarrow \hat{p}_i.
$$

其中，$\mathcal{R}_i$ 说明模型参考的训练集相似案例；$c_i$ 说明 LLM 给出的结构化认知判断，包括情感分布、VAD 状态、模态提示和质量信息；$\beta_i$ 说明文本、语音、视觉和认知推理四类信息源的可靠性分配；$\eta_i$ 说明 LLM 认知信息对基础表示的修正幅度；$\hat{p}_i$ 是最终情感分布。

这种解释链条具有三个层次。第一是证据层，研究者可以查看 retrieved examples，判断模型参考的训练案例是否合理。第二是认知层，研究者可以检查 LLM 输出的 VAD appraisal、modality hint 和 confidence 是否符合当前话语。第三是融合层，研究者可以分析 reliability weights 和 residual alpha，判断模型最终是否真正采纳了 LLM 认知信息。相比单纯输出自然语言解释，该链条与模型实际计算过程一致，因此更适合作为机制可解释性分析依据。

## 4.5 LLM Backbone 与规模分析

为进一步验证本文框架是否依赖特定 LLM，可设计两组 LLM cache 对比实验。第一组为 Qwen2.5 系列规模分析，比较 Qwen2.5-1.5B-Instruct、Qwen2.5-3B-Instruct、Qwen2.5-7B-Instruct 和 Qwen2.5-14B-Instruct。该组实验回答同一模型家族下，认知推理质量是否随模型规模变化。第二组为同规模跨模型比较，比较 Qwen2.5-7B-Instruct、Meta-Llama-3-8B-Instruct、Mistral-7B-Instruct-v0.3 和 InternLM3-8B-Instruct。该组实验回答本文方法是否依赖单一 LLM backbone。

所有 LLM backbone 实验保持 RAG 参数和训练超参数一致，只改变 `--model-path` 和输出 cache 文件名。训练阶段只替换 `--llm-cache`，因此性能差异主要来自不同 LLM 生成的结构化认知变量，而非主模型训练设置变化。

## 4.6 RAG、VAD 与解释性分析

RAG 分析建议比较 MM-RAG、Text-only RAG 和 No-RAG/LLM-only。MM-RAG 同时考虑文本语义和音视频统计状态，Text-only RAG 用于检验音视频状态检索的额外贡献，No-RAG/LLM-only 用于观察缺少训练集证据约束时 LLM 是否更容易偏离数据集标签体系。

VAD 分析可以从两个层面进行。第一，设置 `vad-contrast-weight=0` 去掉 VAD 对比学习，观察连续情感语义约束对性能的影响。第二，屏蔽认知特征中的 appraisal 和 VAD guidance，观察可靠性门控是否仍能形成合理模态权重。前者当前代码可直接运行，后者需要额外加入特征屏蔽开关。

解释性分析建议统计四类可靠性权重 $\beta^t,\beta^a,\beta^v,\beta^c$ 和残差强度 $\eta$ 的整体分布、类别条件分布以及高/低 RAG 质量样本上的差异。预期上，高唤醒类别如 excited、angry 更依赖语音线索，neutral 等语义稳定类别更依赖文本线索；当检索质量和 LLM 置信度较高时，认知分支权重和残差强度应相应增大。这些统计可以直接支撑本文关于机制可解释性的主张。

# 结论

本文针对多模态会话情感识别中融合过程难解释、模态贡献难追踪和 LLM 推理难控制的问题，提出一种证据约束的多模态认知残差融合框架。该框架以 TEN 为多模态骨干，通过多模态 RAG 检索训练集内部相似案例，为 LLM 情感推理提供证据约束；同时将 LLM 输出解析为情感软分布、VAD 评价、模态提示、置信度和检索质量等结构化认知变量，并通过可靠性门控和质量感知残差机制注入多模态表示。实验结果表明，本文方法在 IEMOCAP 和 MELD 上保持有竞争力的识别性能，同时能够输出可追溯的证据链、模态可靠性和认知注入强度。未来工作可进一步扩展到更多 LLM backbone、缺失模态场景以及基于可靠性权重的交互式情感解释。
