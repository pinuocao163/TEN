# 摘要

多模态会话情感识别旨在结合文本语义、语音韵律、视觉表情以及对话上下文，识别话语级情感状态。现有基于 Transformer 的方法能够有效建模跨模态交互，但其融合过程通常依赖隐式神经表示，难以解释模型为何在特定语境下信任某一模态，也难以追溯最终情感判断的证据来源。为提升模型的机制可解释性，本文在 TEN 多模态 Transformer 框架基础上，提出一种多模态 RAG 增强的认知残差融合方法。具体而言，本文首先从训练集中检索与当前话语在文本语义和音视频状态上相似的样本，为大语言模型推理提供数据集内部证据约束；随后，大语言模型生成结构化认知变量，包括情感软分布、VAD 情感评价、模态提示、推理置信度和检索质量；进一步地，模型根据这些认知变量动态估计文本、语音、视觉和认知推理四类信息源的可靠性，并通过质量感知的残差机制控制大语言模型认知信息对 TEN 基础表示的修正幅度。与此同时，本文引入 VAD 引导的对比学习约束，使模型表示空间与情感类别的连续语义结构保持一致。该方法将传统黑箱式多模态融合转化为“检索证据—认知评价—可靠性分配—残差校正”的可解释决策过程。实验在 IEMOCAP 和 MELD 数据集上进行，结果表明，本文方法能够在保持多模态情感识别性能的同时，提高模型决策过程的可追溯性、模态可解释性和预测可信度。

# 引言

多模态会话情感识别旨在根据对话中的文本内容、语音韵律、视觉表情以及上下文交互关系，识别每个话语所表达的情感状态。该任务是情感计算和自然语言处理中的重要研究方向，在人机交互、智能客服、心理健康分析和社交媒体理解等场景中具有广泛应用价值。与单句情感分类不同，会话情感识别不仅需要理解当前话语本身，还需要结合对话历史、说话人状态变化以及不同模态之间的互补或冲突信息。已有研究从说话人关系、图结构上下文、情绪转移和多方对话建模等角度提升会话级情感理解能力 [1][22][34][53]。例如，同一句文本在不同语音语调下可能表达完全不同的情绪，而视觉表情也可能受到遮挡、个体表达习惯或上下文语境的影响。因此，如何有效融合多模态信息，并在复杂会话语境中建模情感变化，是该任务面临的核心挑战。

近年来，基于 Transformer、图神经网络和提示学习的多模态会话情感识别方法取得了显著进展。这类方法通常通过模态内编码器建模单一模态的上下文依赖，并通过模态间注意力机制捕获文本、语音和视觉之间的交互关系。MPT 等方法将多模态提示和对比学习引入 Transformer 框架 [70]，DQ-Former 通过动态模态优先级刻画不同样本中模态贡献的差异 [76]，VEGA 则利用视觉原型增强多模态情感语义对齐 [47]。此外，超图、关系图和跨模态语义图也被用于建模复杂会话结构和模态交互 [39][43][45][51]。然而，现有方法大多仍依赖端到端训练得到的隐式融合表示，其模态权重和决策依据主要隐藏在高维神经空间中。当模型面对模态冲突、弱视觉信号或语音噪声时，往往难以解释其为何信任某一模态，也难以追溯最终情感判断所依据的具体证据。这种机制不透明性限制了模型在高可信人机交互场景中的应用。

另一方面，大语言模型展现出较强的语义理解、上下文推理和情感解释能力，为提升会话情感识别的可解释性提供了新的可能。近期工作已经尝试利用 LLM 建模说话人特征、语音细节、心理状态和对话场景信息，以增强 ERC/MERC 中的高层语义理解能力 [16][31][41][46]。知识增强方法也表明，情绪常识、情绪转移知识、人格信息和情绪原因能够为复杂会话中的情感判断提供重要补充 [6][7][20][58]。相比传统神经分类器，大语言模型能够根据对话上下文生成更接近人类认知过程的情感分析结果，例如判断情感倾向、解释语义线索、识别模态提示以及评估情绪强度。然而，直接将大语言模型用于情感识别仍存在明显风险。首先，大语言模型的推理可能过度依赖通用知识，而不一定符合特定数据集的标签体系和标注风格。其次，大语言模型生成的解释并不必然忠实参与模型预测过程，若仅作为后验说明，仍无法解决神经模型内部决策不可解释的问题。最后，当输入信息不足或检索证据较弱时，大语言模型可能生成不稳定甚至错误的推理结果。因此，关键问题并不是简单地将大语言模型接入多模态情感识别，而是如何使其推理受到任务数据约束，并以可控方式参与神经模型决策。

为解决上述问题，本文提出一种面向机制可解释性的多模态 RAG 增强认知残差融合框架。本文以 TEN 作为基础多模态感知骨干，用于捕获文本、语音和视觉之间的细粒度交互关系；同时引入多模态检索增强生成机制，从训练集中检索与当前话语在文本语义和音视频统计状态上相似的样本，为大语言模型推理提供数据集内部证据约束。与现有的提示学习、知识增强和 LLM 适配方法相比 [8][21][25][31][41]，本文并不直接采用大语言模型输出作为预测结果，而是将其转化为结构化认知变量，包括情感软分布、VAD 情感评价、模态提示、推理置信度和检索质量。随后，模型将这些认知变量作为可靠性建模的依据，用于动态估计文本、语音、视觉和认知推理四类信息源的贡献权重。进一步地，本文设计认知残差校正机制，根据推理置信度和检索质量控制大语言模型认知信息对 TEN 基础表示的修正幅度，从而避免低质量外部推理无条件覆盖神经网络判断。

本文方法的核心思想是将多模态会话情感识别从隐式特征融合问题转化为可追溯的认知决策过程。模型的最终预测不再仅由不可解释的融合向量产生，而是经过“检索证据—认知评价—可靠性分配—残差校正”的逐步推理链条得到。检索样本说明模型参考了哪些相似情感案例，VAD 评价刻画当前话语的连续情感属性，模态提示和可靠性权重说明模型为何信任某些信息源，残差强度则反映大语言模型推理在多大程度上影响最终预测。由此，本文方法不仅增强了模型对复杂多模态情感线索的建模能力，也提升了模型决策过程的可解释性和可信度。

本文的主要贡献总结如下：

1. 提出多模态 RAG 增强的认知推理机制。本文从训练集中联合检索文本语义相似和音视频状态相似的样本，使大语言模型推理受到数据集内部证据约束，从而降低其与目标任务标签体系不一致的风险。
2. 构建结构化认知变量以增强机制可解释性。本文将大语言模型输出表示为情感软分布、VAD 评价、模态提示、推理置信度和检索质量，使其不再只是外部预测器，而是可量化、可监督、可参与融合的认知解释信号。
3. 设计可靠性感知的认知残差融合方法。本文根据认知变量动态估计文本、语音、视觉和大语言模型推理四类信息源的可靠性，并通过质量感知残差机制控制认知信息对 TEN 表示的修正强度，从而实现大语言模型推理与神经多模态表示的稳健融合。
4. 引入 VAD 引导的情感语义约束。本文利用 valence-arousal-dominance 空间刻画情感类别之间的连续语义关系，并结合已有对比学习类方法对情感表示空间的约束思想 [13][25][30][70]，使模型不仅学习离散情感标签，也保持情感语义结构的一致性。

综上，本文提出的方法在保留 TEN 对细粒度多模态交互建模能力的基础上，进一步引入检索增强认知推理和质量可控的残差融合机制，使多模态会话情感识别模型具备更强的证据可追溯性、模态可解释性和预测可信度。

# 相关工作

## 2.1 上下文与图结构会话情感识别

会话情感识别的核心在于建模上下文依赖、说话人交互和情绪状态变化。早期多模态或文本会话方法多从序列上下文出发，随后研究逐渐转向图结构建模，以更精细地表示话语之间、说话人之间以及情绪转移之间的关系。DualGATs 通过双图注意力网络分别建模 discourse-aware graph 和 speaker-aware graph，以捕获会话结构和说话人依赖 [1]。Curriculum Learning Meets DAG 将有向无环图引入多模态情感识别，并通过课程学习逐步提升复杂样本建模能力 [22]。Adaptive Graph Learning 进一步从数据中自适应学习多模态会话图结构，缓解固定图结构难以适配不同对话模式的问题 [34]。

近期图结构方法更关注高阶关系、频谱特征和复杂交互。Graph Spectrum 从图频谱角度重新审视多模态会话情感识别，说明不同频率成分对情感识别具有不同作用 [38]。Dynamic Interactive Bimodal Hypergraph Networks 使用超图建模双模态高阶交互关系 [39]，Hybrid Relational Graphs 则结合情感语义对齐构建混合关系图，以强化多模态语义一致性 [43]。MATCH 通过模态校准的超图融合网络建模会话中复杂的模态关系 [45]，LineConGraphs 从 line conversation graph 角度提升图神经网络的有效性 [62]。这些工作表明，结构化上下文建模已成为 ERC/MERC 的重要方向，但多数方法仍主要从神经特征或图拓扑内部学习关系，缺少外部可追溯证据和显式认知变量。

## 2.2 多模态融合与可靠性建模

多模态会话情感识别需要综合文本语义、语音韵律和视觉表情，不同模态在不同样本中的可靠性并不一致。CMCF-SRNet 通过跨模态上下文融合和语义细化提升模态交互质量 [3]，MultiEMO 利用相关性感知注意力框架建模多模态融合关系 [4]。Joyful 将联合模态融合与图对比学习结合起来，以增强跨模态情感表示 [17]。MPT 通过多模态提示 Transformer 和混合对比学习提升不同模态之间的语义协同 [70]，而 Revisiting Disentanglement and Fusion 则从模态与上下文解耦角度重新分析 MERC 中的融合机制 [71]。

近年来，模态可靠性、模态平衡和动态优先级成为多模态融合的重要问题。Multimodal Emotion Recognition Calibration in Conversations 关注多模态情感识别中的校准问题，使模型输出更加可信 [73]。Ada2I 试图增强不同模态之间的平衡，缓解某一模态过强或过弱带来的融合偏差 [74]。Multimodal Fusion via Hypergraph Autoencoder and Contrastive Learning 结合超图自编码器和对比学习建模跨模态关系 [75]。DQ-Former 使用动态模态优先级机制，使模型能够根据认知对齐需求选择更可靠的信息源 [76]。DRKF 则通过表示解耦和知识融合增强多模态情感识别 [77]。与这些方法相比，本文不仅学习模态融合权重，还将 LLM 生成的模态提示、置信度和检索质量作为可靠性分配的显式依据，从而增强融合过程的可解释性。

## 2.3 知识增强、提示学习与 LLM 情感推理

会话情感往往依赖隐含常识、情绪原因、说话人特征和情绪转移规律，仅依靠端到端特征学习容易忽略深层认知线索。ECoK 通过情绪常识知识图谱挖掘情感相关知识 [6]，EmoTransKG 利用情绪知识图谱揭示情绪转移规律 [7]。SKIER 将符号知识融入会话情感识别 [32]，MKE-IGN 通过多知识增强交互图网络融合外部知识与多模态上下文 [21]。UniMEEC 将多模态情感识别和情感原因分析统一建模，使情感类别判断与原因理解相互促进 [20]。

随着大语言模型发展，LLM 和提示学习方法逐渐被引入 ERC/MERC。Multimodal Prompt Learning with Missing Modalities 关注缺失模态场景下的提示学习问题 [8]，CEPT 通过对比增强提示调优提升会话情感识别效果 [25]。MSE-Adapter 设计轻量插件，使 LLM 获得多模态情感分析和情感识别能力 [41]。LaERC-S 利用 LLM 挖掘说话人特征以增强 ERC [31]，Beyond Silent Letters 进一步强调语音细节对 LLM 情感理解的重要性 [16]。DialogueMMT 使用对话场景理解增强的多模态多任务调优框架，提升多模态会话情感识别表现 [11]。这些方法证明 LLM 和提示学习可以提供高层语义补充，但仍需要解决生成推理与数据集标签体系之间的一致性问题。本文通过训练集内部多模态检索约束 LLM 推理，并将其输出转化为可监督、可筛选、可残差注入的认知变量。

## 2.4 情感语义约束、缺失模态与可信 MERC

除准确率之外，MERC 还需要面对类别语义相近、模态缺失、样本不均衡和模型可信度不足等问题。Emotion-Anchored Contrastive Learning 使用情感锚点对比学习约束表示空间 [13]，Dual Contrastive Learning Framework 通过双重对比学习增强多模态会话情感表示 [30]。PEMV 通过近邻情绪均值向量改善情感空间分布 [15]，Graph Spectrum 也从图信号视角分析情感类别之间的结构差异 [38]。这些方法与本文的 VAD 引导对比学习目标一致，均试图让模型表示不仅区分离散标签，也保留情感类别之间的连续语义关系。

在鲁棒性和可信度方面，缺失模态和说话人泛化是近期重要问题。Multimodal Prompt Learning with Missing Modalities 和 Hardness-Aware Dynamic Curriculum Learning 均关注缺失模态下的稳健情感识别 [8][48]，Contrastive Learning Based Modality-Invariant Feature Acquisition 则从模态不变特征获取角度提升缺失模态鲁棒性 [57]。Generalizing to Unseen Speakers 和 Progressive Contrastive Deep Supervision 关注未知说话人场景中的泛化能力 [61][64]。此外，MERC calibration 和 Dynamic Causal Disentanglement Model 分别从预测校准与因果解耦角度提升模型可信度 [73][66]。本文关注的问题与上述方向互补：通过 RAG 检索证据、LLM 结构化认知变量、可靠性门控和质量感知残差，使模型在多模态融合过程中形成更清晰的证据链和更可控的信息注入机制。

# 方法论

## 3.1 问题定义

本文面向多模态会话情感识别任务。给定一个包含 $N$ 个话语的对话序列：

$$
D=\{u_1,u_2,\ldots,u_N\},
$$

其中，$D$ 表示完整对话，$u_i$ 表示第 $i$ 个话语。每个话语同时包含文本、语音和视觉三类模态特征：

$$
u_i=\{x_i^t,x_i^a,x_i^v\},
$$

其中，$x_i^t$、$x_i^a$ 和 $x_i^v$ 分别表示文本、语音和视觉特征。模型目标是预测每个话语的情感类别：

$$
\hat{y}_i=f(x_i^t,x_i^a,x_i^v,D),\quad \hat{y}_i\in\{1,\ldots,C\},
$$

其中，$C$ 表示数据集中的情感类别数。本文实验涉及 IEMOCAP 和 MELD 两个数据集，其中 IEMOCAP 包含 happy、sad、neutral、angry、excited 和 frustrated 六类情感，MELD 包含 neutral、surprise、fear、sadness、joy、disgust 和 anger 七类情感。

为建模会话中的上下文依赖、说话人状态变化和跨模态互补关系，本文以 TEN 多模态 Transformer 编码器为骨干网络，并进一步引入多模态 RAG 增强的大语言模型认知推理分支。与传统端到端多模态融合不同，本文不直接使用大语言模型结果替代神经网络预测，而是将其转化为可解释的认知变量，并通过可靠性感知融合和认知残差机制，将其以质量受控的方式注入 TEN 表征。

## 3.2 多模态特征编码

对于每个话语 $u_i$，模型首先读取预处理后的文本、语音和视觉特征。项目实现中，文本特征维度为 1024，视觉特征采用 denseface 表征，维度为 342；语音特征在 IEMOCAP 上采用 IS10 特征，维度为 1582，在 MELD 上采用 MELD audio 特征，维度为 300。由于不同模态特征维度不同，模型使用一维卷积将其映射到统一隐空间：

$$
h_i^m=\phi_m(x_i^m),\quad m\in\{t,a,v\},
$$

其中，$h_i^t$、$h_i^a$ 和 $h_i^v$ 分别表示文本、语音和视觉的隐表示，$\phi_m(\cdot)$ 表示对应模态的一维卷积投影函数。会话情感识别还依赖话语顺序和说话人身份，因此模型进一步加入位置编码和说话人嵌入：

$$
\tilde{h}_i^m=h_i^m+p_i+s_i,
$$

其中，$p_i$ 表示位置编码，$s_i$ 表示第 $i$ 个话语的说话人嵌入。项目中 IEMOCAP 设置为 2 个说话人，MELD 设置为 9 个说话人。

## 3.3 模态内与模态间 Transformer 编码

为同时建模单模态内部依赖和跨模态交互，本文采用 TEN 的多路径 Transformer 编码结构。对于三类模态，模型构造如下编码分支：

$$
z_i^{m\leftarrow n}=\operatorname{Transformer}_{m\leftarrow n}(\tilde{h}_i^m,\tilde{h}_i^n),
\quad m,n\in\{t,a,v\}.
$$

当 $m=n$ 时表示模态内编码，当 $m\ne n$ 时表示模态间交互。项目代码中对应九个 Transformer 分支：`t_t`、`a_t`、`v_t`、`a_a`、`t_a`、`v_a`、`v_v`、`t_v` 和 `a_v`。对于任一目标模态 $m$，其增强表示可统一写为：

$$
r_i^m=W_m[z_i^{m\leftarrow t};z_i^{m\leftarrow a};z_i^{m\leftarrow v}].
$$

随后，模型通过门控融合得到基础多模态表示：

$$
h_i^{base}=\sum_{m\in\{t,a,v\}}\alpha_i^m r_i^m,\quad
\alpha_i^m=\frac{\exp(g_i^m)}{\sum_{n\in\{t,a,v\}}\exp(g_i^n)}.
$$

基础融合表示能够捕获多模态互补信息，但其权重主要由隐空间学习得到，仍难以解释模型为何在某一具体样本中信任某一模态。因此，本文进一步引入检索增强认知推理和可靠性感知融合机制。

## 3.4 多模态 RAG 增强认知推理

为使模型预测具有可追溯证据，本文从训练集中检索与当前话语相似的样本。目标话语 $u_i$ 与候选话语 $u_j$ 的相似度定义为：

$$
s(u_i,u_j)=\lambda s_{text}(u_i,u_j)+(1-\lambda)s_{av}(u_i,u_j),
$$

其中，$s_{text}$ 表示文本上下文相似度，$s_{av}$ 表示由语音和视觉统计特征计算得到的多模态相似度，$\lambda$ 为文本相似度权重。模型选取得分最高的 $K$ 个样本作为检索证据：

$$
\mathcal{R}_i=\operatorname{TopK}_{u_j\in\mathcal{D}_{train}}s(u_i,u_j).
$$

基于当前话语、对话上下文和检索集合 $\mathcal{R}_i$，大语言模型生成结构化认知向量：

$$
c_i=[p_i^{llm};v_i^{vad};q_i^{mod};\gamma_i;\rho_i;g_i^{vad}],
$$

其中，$p_i^{llm}$ 表示大语言模型输出的情感软分布；$v_i^{vad}$ 对应项目代码中的 `appraisal` 字段，包含 valence、arousal、dominance 和 certainty 四个维度；$q_i^{mod}$ 表示文本、语音和视觉的模态提示；$\gamma_i$ 表示推理置信度；$\rho_i$ 表示检索质量；$g_i^{vad}$ 表示由 VAD 评价派生出的三维情感指导向量。项目中该认知向量维度为 $C+12$，其中 $C$ 为数据集情感类别数。该认知向量并不直接作为最终预测，而是作为后续可靠性建模的依据。

## 3.5 可靠性感知认知残差融合

首先，模型将认知向量 $c_i$ 投影到与 TEN 表示一致的隐空间：

$$
h_i^c=\psi(c_i).
$$

随后，模型计算文本、语音、视觉和认知推理四类信息源的可靠性权重：

$$
\beta_i=\operatorname{softmax}(W_r[r_i^t;r_i^a;r_i^v;h_i^c]+b(c_i)),
$$

其中，$\beta_i=[\beta_i^t,\beta_i^a,\beta_i^v,\beta_i^c]$ 表示可靠性分布，$b(c_i)$ 为由模态提示、VAD 评价、推理置信度和检索质量共同生成的认知偏置项。若某个话语没有通过 LLM cache 覆盖或质量阈值过滤，则代码会对认知分支权重进行 mask，使其不参与该话语的认知融合。基于该可靠性分布，得到认知增强表示：

$$
h_i^{rel}=\beta_i^t r_i^t+\beta_i^a r_i^a+\beta_i^v r_i^v+\beta_i^c h_i^c.
$$

为防止低质量检索或不可靠大语言模型推理直接干扰最终预测，本文不是简单地将认知表示加到基础表示上，而是采用插值式残差校正：

$$
h_i^{final}=h_i^{base}+\eta_i(h_i^{rel}-h_i^{base}),
$$

其中，$\eta_i$ 为认知残差强度，用于控制认知推理对基础表示的修正幅度。当 $\eta_i$ 接近 0 时，模型主要保留 TEN 基础融合表示；当 $\eta_i$ 增大时，模型向可靠性感知的认知增强表示靠近。该写法与项目代码中的 `all_transformer_out + reasoning_alpha * (reasoning_out - all_transformer_out)` 保持一致。残差强度由全局可学习初始值、推理置信度和检索质量共同决定：

$$
\eta_i=\sigma(\theta_\eta)\cdot(0.5+\sigma(w_\eta[\gamma_i;\rho_i]+b_\eta))\cdot\gamma_i(0.5+0.5\rho_i).
$$

最终情感预测为：

$$
\hat{p}_i=\operatorname{softmax}(W_o h_i^{final}+b_o).
$$

## 3.6 训练目标

设一个 batch 中所有有效话语的索引集合为：

$$
\Omega=\{i\mid m_i=1\},
$$

其中，$m_i$ 为 mask 标记，$m_i=1$ 表示真实话语，$m_i=0$ 表示 padding。模型总损失定义为：

$$
\mathcal{L}=\mathcal{L}_{ce}+\lambda_{uni}\mathcal{L}_{uni}
+\lambda_{sd}\mathcal{L}_{sd}
+\lambda_{llm}\mathcal{L}_{llm}
+\lambda_{rel}\mathcal{L}_{rel}
+\lambda_{vad}\mathcal{L}_{vad}.
$$

其中，$\mathcal{L}_{ce}$ 为主分类损失，$\mathcal{L}_{uni}$ 为单模态监督损失，$\mathcal{L}_{sd}$ 为自蒸馏损失，$\mathcal{L}_{llm}$ 为大语言模型认知蒸馏损失，$\mathcal{L}_{rel}$ 为可靠性监督损失，$\mathcal{L}_{vad}$ 为 VAD 引导的对比损失。

主分类损失采用交叉熵：

$$
\mathcal{L}_{ce}=-\frac{1}{|\Omega|}\sum_{i\in\Omega}\sum_{k=1}^{C}y_{ik}\log \hat{p}_{ik}.
$$

大语言模型认知蒸馏损失为：

$$
\mathcal{L}_{llm}=\frac{1}{|\Omega|}\sum_{i\in\Omega}w_i
\operatorname{KL}(p_i^{llm}\Vert \hat{p}_i),
$$

其中，$w_i$ 由 LLM mask、质量阈值、推理置信度和检索质量共同确定。只有同时满足 `llm-min-quality` 和 `llm-min-confidence` 的样本才会被赋予有效蒸馏权重。可靠性监督损失为：

$$
\mathcal{L}_{rel}=\frac{1}{|\Omega|}\sum_{i\in\Omega}\|\beta_i-\beta_i^\ast\|_2^2,
$$

其中，$\beta_i^\ast$ 表示由模态提示、VAD 评价、置信度和检索质量构造的目标可靠性分布。项目实现中，文本、语音、视觉三类目标权重由 LLM 模态提示和 VAD 派生模态提示加权得到，认知分支目标权重由置信度、检索质量和 VAD 强度共同控制。VAD 引导的对比损失用于约束表示空间：同标签样本的正样本权重由 VAD 相似度调节；在 IEMOCAP 上，happy/excited、angry/frustrated、neutral/sad 被显式作为易混淆负样本对进行间隔约束。

## 3.7 决策过程可解释性

本文方法的解释性来自模型内部计算链条，而非预测后的外部说明。对于话语 $u_i$，检索集合 $\mathcal{R}_i$ 表示模型参考的相似样本，认知向量 $c_i$ 表示大语言模型给出的情感分布、VAD 状态和模态提示，可靠性权重 $\beta_i$ 表示文本、语音、视觉和认知推理四类信息源的贡献比例，残差强度 $\eta_i$ 表示认知推理对原始 TEN 表示的修正程度。因此，模型形成如下解释链条：

$$
\mathcal{R}_i\rightarrow c_i\rightarrow \beta_i\rightarrow \eta_i\rightarrow \hat{p}_i.
$$

该链条使情感预测能够从最终类别追溯到检索证据、认知评价、模态可靠性和残差校正过程，从而提升多模态会话情感识别的机制可解释性与可信度。

## 4.2 实现细节

基础骨干网络采用 TEN 多模态 Transformer。模型首先通过一维卷积将文本、语音和视觉特征映射到统一的 1024 维隐空间，然后通过九条模态内与模态间 Transformer 分支建模文本、语音和视觉之间的上下文交互关系。所有实验均使用 Adam 优化器，权重衰减为 1e-5，dropout 为 0.5，Transformer 注意力头数为 8，训练轮数为 150。当前项目运行脚本中，IEMOCAP 使用 `train.py`，学习率为 1e-4，batch size 为 16，自蒸馏温度为 1，并在主分类损失中使用类别权重以缓解类别不均衡；MELD 使用 `train_1.py`，学习率为 5e-6，batch size 为 16，自蒸馏温度为 8，主分类损失不额外设置类别权重。

## 4.5 消融实验分析

A0 与 A1 的比较用于衡量 RAG-LLM 认知分支是否能在不依赖额外监督项的情况下改善 TEN 表示。若 A1 优于 A0，说明结构化认知变量经过残差融合后能够为多模态表示提供有效补充；若提升有限，则说明仅引入认知表示不足以稳定改善性能，需要进一步通过蒸馏或可靠性监督约束其作用方式。

A1 与 A2 的比较用于验证 LLM 情感软分布蒸馏的贡献。LLM 软分布不仅提供单一情感标签，还包含类别间相似性信息，因此能够缓解 hard label 监督过于离散的问题。A2 与 A3 的比较用于验证可靠性监督的作用，即模型是否能够根据 VAD、模态提示、置信度和检索质量学习更合理的 text/audio/visual/LLM 权重分配。A3 与 A5 的比较用于验证 VAD-aware contrastive learning 是否能够进一步约束表示空间，使同类样本在 VAD 语义空间中更加紧凑，并拉开易混淆情感对。

A6 用于检验质量门控的重要性。当 llm-min-quality 和 llm-min-confidence 均设为 0 时，所有 LLM 推理结果都会进入融合过程。若 A6 性能下降，说明低质量检索或低置信度推理会给模型带来噪声，质量门控能够提高认知信息注入的稳健性。A7 用于检验残差强度的作用。当 residual init 设置为 0.0001 时，LLM 认知分支对 TEN 表示的修正幅度被显著压低；若 A7 低于完整模型，说明质量感知认知残差校正是性能提升的重要来源。

## 4.6 RAG 与 VAD 进一步分析

为进一步分析检索增强机制，可以比较多模态 RAG、仅文本 RAG 和无 RAG 的 LLM-only 三种 cache。多模态 RAG 同时考虑文本语义相似度和音视频统计状态相似度，能够使 LLM 推理更贴近数据集内部标注风格；仅文本 RAG 可以检验音视频状态检索的额外贡献；无 RAG 设置用于观察缺少数据集内部证据约束时 LLM 推理是否更容易偏离目标标签体系。

VAD 分析可以从两个层面展开。第一，设置 vad-contrast-weight=0，去掉 VAD 引导对比学习，观察情感语义结构约束对最终性能的影响。第二，将认知向量中的 VAD/appraisal 信息置零，观察可靠性门控是否仍能有效分配模态权重。前者当前代码可直接运行，后者需要额外加入参数开关以屏蔽 llm_features 中对应切片。

## 4.7 可解释性分析

本文方法的解释性来自模型内部计算链条，而不是预测后的外部说明。对于每个话语，RAG 检索样本给出数据集内部证据，LLM 结构化输出给出情感软分布、VAD 评价和模态提示，可靠性门控给出 text/audio/visual/LLM 四类信息源的贡献权重，残差强度 $\eta$ 则表示认知推理对 TEN 基础表示的修正幅度。

实验中建议统计四类可靠性权重的均值与类别条件分布。例如，在 excited 或 angry 等高唤醒情感中，audio 权重通常更应得到提升；在 neutral 等语义较稳定的类别中，text 权重可能更高；当 RAG 检索质量和 LLM 置信度较高时，LLM 认知分支权重及残差强度 $\eta$ 应相应增大。通过这些统计，可以验证模型是否真正形成了“检索证据—认知评价—可靠性分配—残差校正”的可追溯决策过程。

# 结论

本文面向多模态会话情感识别中融合过程难解释、模态贡献难追踪的问题，提出了一种多模态 RAG 增强的认知残差融合方法。该方法以 TEN 为基础骨干，通过检索相似情感样本为大语言模型推理提供证据约束，并将其生成的情感软分布、VAD 评价、模态提示和质量信息转化为可控的认知变量，用于指导可靠性感知融合与残差校正。与此同时，VAD 引导的对比约束进一步增强了表示空间的情感语义一致性。实验结果表明，本文方法能够在提升或保持情感识别性能的同时，使模型决策过程具备更好的证据可追溯性、模态可解释性和预测可信度。
