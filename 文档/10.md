# 摘要

多模态会话情感识别旨在结合文本语义、语音韵律、视觉表情以及对话上下文，识别话语级情感状态。现有基于 Transformer 的方法能够有效建模跨模态交互，但其融合过程通常依赖隐式神经表示，难以解释模型为何在特定语境下信任某一模态，也难以追溯最终情感判断的证据来源。为提升模型的机制可解释性，本文在 TEN 多模态 Transformer 框架基础上，提出一种多模态 RAG 增强的认知残差融合方法。具体而言，本文首先从训练集中检索与当前话语在文本语义和音视频状态上相似的样本，为大语言模型推理提供数据集内部证据约束；随后，大语言模型生成结构化认知变量，包括情感软分布、VAD 情感评价、模态提示、推理置信度和检索质量；进一步地，模型根据这些认知变量动态估计文本、语音、视觉和认知推理四类信息源的可靠性，并通过质量感知的残差机制控制大语言模型认知信息对 TEN 基础表示的修正幅度。与此同时，本文引入 VAD 引导的对比学习约束，使模型表示空间与情感类别的连续语义结构保持一致。该方法将传统黑箱式多模态融合转化为“检索证据—认知评价—可靠性分配—残差校正”的可解释决策过程。实验在 IEMOCAP 和 MELD 数据集上进行，结果表明，本文方法能够在保持多模态情感识别性能的同时，提高模型决策过程的可追溯性、模态可解释性和预测可信度。

# 引言

多模态会话情感识别旨在根据对话中的文本内容、语音韵律、视觉表情以及上下文交互关系，识别每个话语所表达的情感状态。该任务是情感计算和自然语言处理中的重要研究方向，在人机交互、智能客服、心理健康分析和社交媒体理解等场景中具有广泛应用价值。与单句情感分类不同，会话情感识别不仅需要理解当前话语本身，还需要结合对话历史、说话人状态变化以及不同模态之间的互补或冲突信息。例如，同一句文本在不同语音语调下可能表达完全不同的情绪，而视觉表情也可能受到遮挡、个体表达习惯或上下文语境的影响。因此，如何有效融合多模态信息，并在复杂会话语境中建模情感变化，是该任务面临的核心挑战。

近年来，基于 Transformer 的多模态会话情感识别方法取得了显著进展。这类方法通常通过模态内编码器建模单一模态的上下文依赖，并通过模态间注意力机制捕获文本、语音和视觉之间的交互关系。TEN 等代表性方法进一步利用多路径 Transformer 结构对不同模态之间的双向关系进行建模，从而增强了模型对跨模态互补信息的捕获能力。然而，现有方法大多仍依赖端到端训练得到的隐式融合表示，其模态权重和决策依据主要隐藏在高维神经空间中。当模型面对模态冲突、弱视觉信号或语音噪声时，往往难以解释其为何信任某一模态，也难以追溯最终情感判断所依据的具体证据。这种机制不透明性限制了模型在高可信人机交互场景中的应用。

另一方面，大语言模型展现出较强的语义理解、上下文推理和情感解释能力，为提升会话情感识别的可解释性提供了新的可能。相比传统神经分类器，大语言模型能够根据对话上下文生成更接近人类认知过程的情感分析结果，例如判断情感倾向、解释语义线索、识别模态提示以及评估情绪强度。然而，直接将大语言模型用于情感识别仍存在明显风险。首先，大语言模型的推理可能过度依赖通用知识，而不一定符合特定数据集的标签体系和标注风格。其次，大语言模型生成的解释并不必然忠实参与模型预测过程，若仅作为后验说明，仍无法解决神经模型内部决策不可解释的问题。最后，当输入信息不足或检索证据较弱时，大语言模型可能生成不稳定甚至错误的推理结果。因此，关键问题并不是简单地将大语言模型接入多模态情感识别，而是如何使其推理受到任务数据约束，并以可控方式参与神经模型决策。

为解决上述问题，本文提出一种面向机制可解释性的多模态 RAG 增强认知残差融合框架。本文以 TEN 作为基础多模态感知骨干，用于捕获文本、语音和视觉之间的细粒度交互关系；同时引入多模态检索增强生成机制，从训练集中检索与当前话语在文本语义和音视频统计状态上相似的样本，为大语言模型推理提供数据集内部证据约束。在此基础上，大语言模型生成结构化认知变量，包括情感软分布、VAD 情感评价、模态提示、推理置信度和检索质量。不同于直接采用大语言模型输出作为预测结果，本文将这些认知变量作为可靠性建模的依据，用于动态估计文本、语音、视觉和认知推理四类信息源的贡献权重。进一步地，本文设计认知残差校正机制，根据推理置信度和检索质量控制大语言模型认知信息对 TEN 基础表示的修正幅度，从而避免低质量外部推理无条件覆盖神经网络判断。

本文方法的核心思想是将多模态会话情感识别从隐式特征融合问题转化为可追溯的认知决策过程。模型的最终预测不再仅由不可解释的融合向量产生，而是经过“检索证据—认知评价—可靠性分配—残差校正”的逐步推理链条得到。检索样本说明模型参考了哪些相似情感案例，VAD 评价刻画当前话语的连续情感属性，模态提示和可靠性权重说明模型为何信任某些信息源，残差强度则反映大语言模型推理在多大程度上影响最终预测。由此，本文方法不仅增强了模型对复杂多模态情感线索的建模能力，也提升了模型决策过程的可解释性和可信度。

本文的主要贡献总结如下：

提出多模态 RAG 增强的认知推理机制。 本文从训练集中联合检索文本语义相似和音视频状态相似的样本，使大语言模型推理受到数据集内部证据约束，从而降低其与目标任务标签体系不一致的风险。

构建结构化认知变量以增强机制可解释性。 本文将大语言模型输出表示为情感软分布、VAD 评价、模态提示、推理置信度和检索质量，使其不再只是外部预测器，而是可量化、可监督、可参与融合的认知解释信号。

设计可靠性感知的认知残差融合方法。 本文根据认知变量动态估计文本、语音、视觉和大语言模型推理四类信息源的可靠性，并通过质量感知残差机制控制认知信息对 TEN 表示的修正强度，从而实现大语言模型推理与神经多模态表示的稳健融合。

引入 VAD 引导的情感语义约束。 本文利用 valence-arousal-dominance 空间刻画情感类别之间的连续语义关系，并通过对比学习约束模型表示空间，使模型不仅学习离散情感标签，也保持情感语义结构的一致性。

综上，本文提出的方法在保留 TEN 对细粒度多模态交互建模能力的基础上，进一步引入检索增强认知推理和质量可控的残差融合机制，使多模态会话情感识别模型具备更强的证据可追溯性、模态可解释性和预测可信度。

# 实验

## 4.1 数据集与评价指标

本文在 IEMOCAP 和 MELD 两个公开多模态会话情感识别数据集上进行实验。IEMOCAP 包含 happy、sad、neutral、angry、excited 和 frustrated 六类情感，测试集共有 1623 个有效话语；MELD 包含 neutral、surprise、fear、sadness、joy、disgust 和 anger 七类情感，测试集共有 2610 个有效话语。两套数据均采用项目中预处理后的文本、语音和视觉特征，其中 IEMOCAP 的文本、视觉和语音特征维度分别为 1024、342 和 1582，MELD 的文本、视觉和语音特征维度分别为 1024、342 和 300。

评价指标采用 Accuracy、Macro-F1 和 Weighted-F1。由于两个数据集均存在类别分布不均衡，本文主要以 Weighted-F1 作为模型整体性能的主要比较指标，同时报告 Accuracy 和 Macro-F1 以反映类别平均性能。

基础骨干网络采用 TEN 多模态 Transformer。模型首先通过一维卷积将文本、语音和视觉特征映射到统一的 1024 维隐空间，然后通过九条模态内与模态间 Transformer 分支建模文本、语音和视觉之间的上下文交互关系。所有实验均使用 Adam 优化器，权重衰减为 1e-5，dropout 为 0.5，Transformer 注意力头数为 8，训练轮数为 150。IEMOCAP 的学习率设为 1e-4，batch size 为 16，蒸馏温度为 1；MELD 的学习率设为 5e-5，batch size 为 16，蒸馏温度为 1。

认知推理分支使用离线生成的 RAG-LLM cache。IEMOCAP 使用 data/iemocap_llm_reasoning_mmrag.jsonl，MELD 使用 data/meld_llm_reasoning_mmrag.jsonl。每个 cache 条目包含情感软分布、VAD/appraisal 评价、模态提示、推理置信度、检索质量和 VAD 指导向量。训练时不直接使用 LLM 输出替代模型预测，而是通过可靠性感知门控和质量感知残差机制将认知信息注入 TEN 表示。

## 4.5 消融实验分析

A0 与 A1 的比较用于衡量 RAG-LLM 认知分支是否能在不依赖额外监督项的情况下改善 TEN 表示。若 A1 优于 A0，说明结构化认知变量经过残差融合后能够为多模态表示提供有效补充；若提升有限，则说明仅引入认知表示不足以稳定改善性能，需要进一步通过蒸馏或可靠性监督约束其作用方式。

A1 与 A2 的比较用于验证 LLM 情感软分布蒸馏的贡献。LLM 软分布不仅提供单一情感标签，还包含类别间相似性信息，因此能够缓解 hard label 监督过于离散的问题。A2 与 A3 的比较用于验证可靠性监督的作用，即模型是否能够根据 VAD、模态提示、置信度和检索质量学习更合理的 text/audio/visual/LLM 权重分配。A3 与 A5 的比较用于验证 VAD-aware contrastive learning 是否能够进一步约束表示空间，使同类样本在 VAD 语义空间中更加紧凑，并拉开易混淆情感对。

A6 用于检验质量门控的重要性。当 llm-min-quality 和 llm-min-confidence 均设为 0 时，所有 LLM 推理结果都会进入融合过程。若 A6 性能下降，说明低质量检索或低置信度推理会给模型带来噪声，质量门控能够提高认知信息注入的稳健性。A7 用于检验残差强度的作用。当 residual init 设置为 0.0001 时，LLM 认知分支对 TEN 表示的修正幅度被显著压低；若 A7 低于完整模型，说明质量感知认知残差校正是性能提升的重要来源。

## 4.6 RAG 与 VAD 进一步分析

为进一步分析检索增强机制，可以比较多模态 RAG、仅文本 RAG 和无 RAG 的 LLM-only 三种 cache。多模态 RAG 同时考虑文本语义相似度和音视频统计状态相似度，能够使 LLM 推理更贴近数据集内部标注风格；仅文本 RAG 可以检验音视频状态检索的额外贡献；无 RAG 设置用于观察缺少数据集内部证据约束时 LLM 推理是否更容易偏离目标标签体系。

VAD 分析可以从两个层面展开。第一，设置 vad-contrast-weight=0，去掉 VAD 引导对比学习，观察情感语义结构约束对最终性能的影响。第二，将认知向量中的 VAD/appraisal 信息置零，观察可靠性门控是否仍能有效分配模态权重。前者当前代码可直接运行，后者需要额外加入参数开关以屏蔽 llm_features 中对应切片。

## 4.7 可解释性分析

本文方法的解释性来自模型内部计算链条，而不是预测后的外部说明。对于每个话语，RAG 检索样本给出数据集内部证据，LLM 结构化输出给出情感软分布、VAD 评价和模态提示，可靠性门控给出 text/audio/visual/LLM 四类信息源的贡献权重，残差强度 alpha 则表示认知推理对 TEN 基础表示的修正幅度。

实验中建议统计四类可靠性权重的均值与类别条件分布。例如，在 excited 或 angry 等高唤醒情感中，audio 权重通常更应得到提升；在 neutral 等语义较稳定的类别中，text 权重可能更高；当 RAG 检索质量和 LLM 置信度较高时，LLM 认知分支权重及残差 alpha 应相应增大。通过这些统计，可以验证模型是否真正形成了“检索证据—认知评价—可靠性分配—残差校正”的可追溯决策过程。

# 结论

本文面向多模态会话情感识别中融合过程难解释、模态贡献难追踪的问题，提出了一种多模态 RAG 增强的认知残差融合方法。该方法以 TEN 为基础骨干，通过检索相似情感样本为大语言模型推理提供证据约束，并将其生成的情感软分布、VAD 评价、模态提示和质量信息转化为可控的认知变量，用于指导可靠性感知融合与残差校正。与此同时，VAD 引导的对比约束进一步增强了表示空间的情感语义一致性。实验结果表明，本文方法能够在提升或保持情感识别性能的同时，使模型决策过程具备更好的证据可追溯性、模态可解释性和预测可信度。
