# 面向可审计多模态情感数据处理的证据锚定认知校准框架

# 摘要

多模态会话情感识别（Multimodal Emotion Recognition in Conversation, MERC）需要在连续对话中联合处理文本语义、语音韵律、视觉表情和说话人上下文，是典型的异构交互数据理解问题。与静态多模态分类不同，MERC 数据在处理流程中同时面临三类瓶颈：不同模态的时间粒度和语义密度不一致，真实场景中的语音和视觉信号容易受到噪声或缺失影响，端到端融合模型难以追踪每一次预测依赖了哪些数据证据。现有 Transformer、图神经网络和提示学习方法虽然提升了整体识别性能，但大多把多模态融合压缩为隐式高维表示，难以回答“检索了什么相似案例、信任了哪个模态、外部认知推理实际改变了多少模型判断”等关键问题。

本文提出证据锚定认知校准框架（Evidence-Anchored Cognitive Calibration, EACC），将 MERC 重构为一条可审计的数据处理流水线：首先从训练集中构建多模态 RAG 证据，以文本上下文相似度和音视频状态相似度共同约束大语言模型推理；然后把 LLM 输出解析为结构化认知 cache，包括情感软分布、VAD appraisal、模态可靠性提示、推理置信度和检索质量；最后通过质量感知可靠性门控，将文本、语音、视觉和认知分支统一到可控融合空间中。该机制既保留 LLM 的高层语义推理能力，又避免低质量外部推理无条件覆盖原始多模态观测。实验在 IEMOCAP 和 MELD 上验证了 EACC 的有效性：相较所选可比基线中的最强结果，EACC 在 IEMOCAP 上将 Weighted-F1 提升至 74.99%，提升 2.44 个百分点，将 Accuracy 提升至 74.86%，提升 1.59 个百分点；在 MELD 上取得 67.62% Weighted-F1 和 68.58% Accuracy，分别提升 0.51 和 0.21 个百分点。消融实验进一步表明，移除结构化认知变量会使 IEMOCAP 和 MELD 的 Weighted-F1 分别下降 1.64 和 1.27，移除质量门控分别下降 0.87 和 0.71，说明本文方法的收益来自证据约束、质量筛选和可靠性校准的协同作用。除识别性能外，EACC 还能够输出检索证据、模态贡献和推理注入强度，为多模态情感数据处理提供可追溯、可控且可解释的决策机制。

# 1 引言

多模态会话情感识别旨在根据对话中的文本内容、语音韵律、视觉表情和上下文交互关系，识别每个话语所表达的情感状态。该任务是情感计算、可信人机交互、智能客服、社交媒体理解和心理健康分析中的基础问题。随着交互式应用不断产生大规模多源数据，MERC 已不只是一个模型分类问题，更是一个复杂的数据工程问题：系统需要从异构模态中抽取稳定信号，在噪声和缺失条件下完成对齐与融合，并在最终预测出错时追溯数据证据和处理路径。

实际 MERC 数据具有明显的流水线难点。首先，文本、语音和视觉的观测形式不同：文本离散且语义密集，语音包含连续韵律变化，视觉则受到表情幅度、遮挡和镜头质量影响。直接把三类特征拼接或用注意力融合，往往会把数据质量差异隐藏在不可解释的权重中。其次，会话情感具有强上下文依赖，同一句话在不同说话人关系、历史轮次和情绪转移下可能对应不同标签。再次，情感类别之间并非彼此孤立，happy 与 excited、sad 与 frustrated 等类别常常只在 valence、arousal 和 dominance 等连续情感维度上存在细粒度差异。已有工作从说话人关系、图结构上下文、情绪转移和多模态融合等角度提升了识别能力 [1][22][34][53]，但多数方法仍难以解释模型在某个样本上为何选择某一模态或某一类情感。

近年来，图神经网络、超图建模、多模态提示学习和可靠性建模显著推动了 MERC 的发展。MPT 将多模态提示和对比学习引入 Transformer 框架 [70]，DQ-Former 通过动态模态优先级刻画样本级模态贡献 [76]，MATCH 使用模态校准超图增强多模态融合 [45]，VEGA 利用视觉原型强化情感语义对齐 [47]。这些方法证明了上下文结构和模态交互的重要性，但它们的证据链主要存在于隐式神经表示内部。当视觉信号低质量、语音韵律被噪声污染或文本存在讽刺时，研究者很难判断模型是否依据了可靠数据，还是仅仅利用训练分布偏置完成预测。

LLM 的上下文理解、情感解释和常识推理能力为 MERC 提供了新的机会。已有研究尝试利用 LLM 建模说话人特征、语音细节、心理状态和对话场景信息 [16][31][41][46]，知识增强方法也表明情绪常识、人格信息、情绪原因和情绪转移知识能够补充神经模型的局限 [6][7][20][58]。然而，直接把 LLM 输出作为分类结果或后验解释并不能解决 MERC 的数据处理瓶颈。一方面，开放域 LLM 可能偏离 IEMOCAP 或 MELD 的标签边界；另一方面，后验解释不一定真实参与模型决策；此外，当检索证据质量较低或上下文信息不足时，LLM 推理本身可能成为新的噪声源。因此，关键问题不是简单接入 LLM，而是如何把 LLM 推理转化为受数据集证据约束、可度量、可过滤、可参与训练的结构化信号。

本文提出 EACC，将 LLM 从外部预测器转化为多模态情感数据处理流水线中的认知 cache 生成器。对于每个目标话语，EACC 首先从训练集中检索文本上下文和音视频状态相似的标注样本，使 LLM 推理对齐数据集内部标签边界；随后把 LLM 输出解析为情感软分布、VAD appraisal、模态提示、置信度和检索质量；最后由质量感知校准模块决定文本、语音、视觉和认知分支各自应被信任多少，以及外部认知信息应该以多大强度修正基础多模态表示。这样，模型的预测结果不再只是黑箱分类，而可以沿着“检索证据—结构化认知—可靠性分配—校准强度—最终预测”的路径被审计。

本文的贡献如下。

1. 提出证据锚定的多模态 RAG 认知 cache 构建机制。该机制联合文本上下文相似度和音视频状态相似度检索训练集样本，用数据集内部证据约束 LLM 推理，解决开放域推理与任务标签边界不一致的问题；实验中的 cache 诊断显示，Qwen2.5-7B 在 IEMOCAP 上达到最高标签一致性 0.5711，为后续认知注入提供了更可靠的数据基础。
2. 设计结构化认知变量与质量感知校准机制。EACC 将 LLM 输出转化为情感软分布、VAD appraisal、模态提示、推理置信度和检索质量，并据此动态估计文本、语音、视觉和认知分支的可靠性，解决多模态融合中数据质量差异不可见、外部推理不可控的问题；移除结构化认知变量后，IEMOCAP 和 MELD 的 Weighted-F1 分别下降 1.64 和 1.27。
3. 构建面向审计的多模态情感决策链。EACC 通过可靠性权重和校准强度显式记录每个样本的证据来源、模态贡献和认知注入幅度，解决高性能模型难解释、难复查的问题；在 IEMOCAP 和 MELD 上，EACC 分别取得 74.99% 和 67.62% Weighted-F1，优于所选可比基线，并能为典型样本输出完整解释链。

# 2 相关工作

## 2.1 上下文与图结构会话情感识别

会话情感识别的核心在于建模上下文依赖、说话人交互和情绪状态变化。DualGATs 通过双图注意力网络分别建模 discourse-aware graph 和 speaker-aware graph [1]，Curriculum Learning Meets DAG 将有向无环图引入多模态情感识别并结合课程学习 [22]，Adaptive Graph Learning 从数据中自适应学习多模态会话图结构 [34]。近期研究进一步关注高阶关系和复杂交互，Graph Spectrum 从图频谱角度重新审视 MERC 中的上下文传播机制 [38]，Dynamic Interactive Bimodal Hypergraph Networks 使用超图建模双模态高阶关系 [39]，Hybrid Relational Graphs 和 MATCH 分别从情感语义对齐和模态校准超图角度增强多模态融合 [43][45]。这些方法说明结构化上下文对 ERC/MERC 至关重要，但多数工作仍主要在神经特征或图拓扑内部传播信息，缺少可被审计的样本级数据证据。

## 2.2 多模态融合与可靠性建模

文本、语音和视觉在不同话语中的可靠性并不相同。CMCF-SRNet 通过跨模态上下文融合和语义细化提升模态交互质量 [3]，MultiEMO 利用相关性感知注意力框架建模多模态关系 [4]，Joyful 将联合模态融合与图对比学习结合起来 [17]。近期研究进一步关注模态平衡、动态优先级和可信输出，MERC Calibration 关注多模态情感识别中的校准问题 [73]，Ada2I 缓解模态不平衡 [74]，DQ-Former 根据样本动态查询不同模态优先级 [76]，DRKF 通过表示解耦和知识融合增强多模态情感识别 [77]。这些方法主要学习模型内部的可靠性权重，而 EACC 进一步把 RAG 证据质量、LLM 置信度、VAD appraisal 和模态提示纳入可靠性分配，使融合权重不仅可学习，而且具有明确的数据来源。

## 2.3 知识增强、提示学习与 LLM 情感推理

会话情感往往依赖隐含常识、情绪原因、说话人特征和情绪转移规律。ECoK 挖掘情绪常识知识图谱 [6]，EmoTransKG 利用情绪知识图谱揭示情绪转移 [7]，SKIER 将符号知识融入会话情感识别 [32]，MKE-IGN 通过多知识增强交互图网络融合外部知识和多模态上下文 [21]，UniMEEC 将情感识别与情感原因分析统一建模 [20]。随着 LLM 发展，提示学习和 LLM 适配方法也被引入 MERC。MSE-Adapter 设计轻量插件赋予 LLM 多模态情感分析能力 [41]，LaERC-S 利用 LLM 挖掘说话人特征 [31]，Beyond Silent Letters 强调语音细节对 LLM 情感理解的重要性 [16]，DialogueMMT 利用对话场景理解增强多模态多任务调优 [11]。与这些方法相比，本文强调 LLM 推理必须被训练集证据约束，并以结构化变量真实进入模型计算链条。EACC 不是把 LLM 作为独立分类器或后验解释器，而是把 RAG 证据、VAD 评价、模态可靠性提示和软标签分布编码成可学习、可筛选、可审计的认知 cache。

# 3 方法

## 3.1 任务定义与数据处理流水线

给定包含 $N$ 个话语的多模态对话 $D=\{u_1,\ldots,u_N\}$，每个话语包含文本、语音和视觉特征 $u_i=\{x_i^t,x_i^a,x_i^v\}$。模型目标是在对话上下文中预测话语级情感标签 $\hat{y}_i\in\{1,\ldots,C\}$，其中 $C$ 是情感类别数，$t,a,v$ 分别表示 text、audio 和 visual 三类模态。

传统 MERC 模型通常直接学习从多模态观测到情感标签的映射：

$$
(x_i^t,x_i^a,x_i^v)\rightarrow h_i^{base}\rightarrow \hat{p}_i,
$$

其中，$h_i^{base}$ 是基础多模态表示，$\hat{p}_i$ 是情感概率分布。该流程虽然高效，但把数据质量、证据来源和融合可靠性都压缩进隐式表示。本文将其扩展为一条可审计的数据处理链：

$$
\mathcal{R}_i \rightarrow c_i,\quad
(h_i^{base},c_i)\rightarrow h_i^{final}\rightarrow \hat{p}_i.
$$

其中，$\mathcal{R}_i$ 是从训练集中检索到的相似证据集合，$c_i$ 是由 RAG-LLM 分支生成的结构化认知变量，$h_i^{final}$ 是经 EACC 校准后的最终表示。EACC 包含四个阶段：三模态交互编码、多模态证据检索、结构化认知 cache 构建、质量受控的证据校准。训练阶段只读取离线 cache，不在线加载 LLM，因此推理成本主要由多模态编码器承担。

## 3.2 三模态交互编码器

对于每个模态 $m\in\{t,a,v\}$，模型首先用一维卷积或线性投影把原始特征映射到统一隐空间：

$$
h_i^m=\phi_m(x_i^m).
$$

其中，$\phi_m(\cdot)$ 是模态 $m$ 的投影函数，$h_i^m$ 是第 $i$ 个话语在该模态上的初始隐表示。为了保留会话顺序和说话人信息，模型加入位置编码 $p_i$ 与说话人嵌入 $s_i$：

$$
\tilde{h}_i^m=h_i^m+p_i+s_i.
$$

随后，模型构建九条 Transformer 交互分支：

$$
z_i^{m\leftarrow n}
=\operatorname{Transformer}_{m\leftarrow n}(\tilde{H}^n,\tilde{H}^m),
\quad m,n\in\{t,a,v\}.
$$

当 $m=n$ 时，该分支建模模态内上下文；当 $m\neq n$ 时，该分支建模源模态 $n$ 对目标模态 $m$ 的跨模态影响。对于目标模态 $m$，模型拼接来自三个源模态的交互结果，并得到增强表示：

$$
e_i^m=W_m[z_i^{m\leftarrow t};z_i^{m\leftarrow a};z_i^{m\leftarrow v}]+b_m.
$$

其中，$e_i^m$ 表示第 $i$ 个话语在目标模态 $m$ 上的增强表示，$W_m$ 和 $b_m$ 是可学习参数。基础多模态表示由门控融合得到：

$$
h_i^{base}=\operatorname{Gate}(e_i^t,e_i^a,e_i^v).
$$

$h_i^{base}$ 是没有引入外部认知 cache 时的核心多模态判断，也是后续质量校准的参照点。EACC 的目标不是替代该表示，而是在证据可靠时对其进行受控修正。

## 3.3 多模态 RAG 证据构建

LLM 的开放域知识并不天然符合 IEMOCAP 或 MELD 的标签体系。为使认知推理受到数据集内部证据约束，本文从训练集中检索与目标话语相似的标注样本。对目标话语 $u_i$ 和候选训练话语 $u_j$，检索得分定义为：

$$
s(u_i,u_j)=\lambda s_{text}(u_i,u_j)+(1-\lambda)s_{av}(u_i,u_j).
$$

其中，$s_{text}$ 是基于上下文 TF-IDF 的文本相似度，$s_{av}$ 是基于语音能量、语音变化幅度和视觉动态统计的音视频状态相似度，$\lambda=0.7$ 控制文本证据与音视频状态证据的相对权重。文本相似度捕获“说了什么”，音视频相似度捕获“如何表达”，二者共同缓解只按表面文本检索导致的证据偏差。

最终选取得分最高的 $K$ 个训练样本作为证据：

$$
\mathcal{R}_i=\operatorname{TopK}_{u_j\in\mathcal{D}_{train}}s(u_i,u_j).
$$

为了衡量证据是否值得被模型采纳，本文定义检索质量分数：

$$
\rho_i=0.35s_i^{top}+0.20s_i^{mean}+0.20\max(s_i^{mm},0)+0.25a_i^{label}.
$$

其中，$s_i^{top}$ 是 top-1 证据的综合相似度，$s_i^{mean}$ 是 top-$K$ 证据的平均综合相似度，$s_i^{mm}$ 是平均音视频相似度，$a_i^{label}$ 是检索样本标签与 LLM 输出标签的一致比例。该分数强调最近邻证据和标签一致性，同时保留整体证据密度与音视频状态一致性。$\rho_i$ 越高，说明检索证据越集中、越稳定，后续认知注入应越可信。

## 3.4 结构化认知 cache

LLM 接收当前话语、局部上下文、音视频统计和检索证据后，输出包含 `label`、`confidence`、`rationale`、`appraisal` 和 `modality_hint` 的 JSON 对象。与自由文本解释相比，JSON 输出可以被稳定解析为结构化 cache，并在训练时直接转换为张量。

第 $i$ 个话语的认知变量定义为：

$$
c_i=[p_i^{llm};a_i;q_i;\gamma_i;\rho_i;g_i].
$$

其中，$p_i^{llm}$ 是 LLM 给出的情感软分布；$a_i=[v_i,\alpha_i,d_i,z_i]$ 是 VAD appraisal，$v_i$、$\alpha_i$、$d_i$ 和 $z_i$ 分别表示 valence、arousal、dominance 和 certainty；$q_i=[q_i^t,q_i^a,q_i^v]$ 是 LLM 对文本、语音、视觉三类模态可靠性的提示；$\gamma_i$ 是 LLM 推理置信度；$\rho_i$ 是检索质量；$g_i$ 是由 VAD appraisal 派生的指导向量。

VAD guidance 定义为：

$$
g_i=[g_i^{pos},g_i^{ctrl},g_i^{low}],
$$

$$
g_i^{pos}=\max(v_i,0)\cdot \alpha_i,
$$

$$
g_i^{ctrl}=\max(-v_i,0)\cdot\max(\alpha_i,0)\cdot d_i,
$$

$$
g_i^{low}=\max(-v_i,0)\cdot\max(-\alpha_i,0).
$$

$g_i^{pos}$ 描述正向高激活状态，$g_i^{ctrl}$ 描述负向高唤醒且带控制感的状态，$g_i^{low}$ 描述负向低唤醒状态。这三类连续情感因素能够补充离散标签，帮助模型区分 happy/excited、angry/frustrated、neutral/sad 等易混淆类别。认知变量的维度为：

$$
\dim(c_i)=C+4+3+1+1+3=C+12.
$$

因此，IEMOCAP 的认知变量维度为 18，MELD 的认知变量维度为 19。

## 3.5 证据锚定认知校准

EACC 的核心是把结构化认知变量转化为两个可解释控制量：可靠性分布和校准强度。可靠性分布回答模型在当前样本上应信任文本、语音、视觉还是认知分支；校准强度回答外部认知信息应在多大程度上修正基础多模态表示。

首先，认知变量被投影到统一隐空间：

$$
h_i^c=\psi(c_i).
$$

其中，$h_i^c\in\mathbb{R}^{d}$ 是认知分支隐表示，$\psi(\cdot)$ 由线性层、非线性激活和 LayerNorm 组成，$d=1024$ 是统一隐空间维度。

模型计算四类信息源的基础可靠性 logits：

$$
o_i=W_r[e_i^t;e_i^a;e_i^v;h_i^c]+b_r.
$$

其中，$o_i\in\mathbb{R}^{4}$ 是文本、语音、视觉和认知分支的未归一化可靠性分数。随后，EACC 引入模态提示偏置：

$$
b_i^{hint}=W_h[q_i^t,q_i^a,q_i^v,\tau_i],
$$

$$
\tau_i=\gamma_i(0.5+0.5\rho_i).
$$

$\tau_i$ 表示 reasoning trust，它由 LLM 置信度 $\gamma_i$ 和检索质量 $\rho_i$ 共同决定。若 LLM 自身置信度高且检索证据质量高，认知分支才具备更强的门控影响力。

同时，模型将 VAD appraisal 和 guidance 转化为情感结构偏置：

$$
b_i^{vad}=W_v[a_i;g_i].
$$

最终可靠性权重为：

$$
\beta_i=\operatorname{softmax}(o_i+b_i^{hint}+b_i^{vad}).
$$

其中，$\beta_i=[\beta_i^t,\beta_i^a,\beta_i^v,\beta_i^c]$，四个分量分别表示文本、语音、视觉和认知分支的可靠性权重。$\operatorname{softmax}$ 保证四个权重非负且和为 1。

为避免低质量 cache 干扰模型，本文使用质量 mask：

$$
M_i^{llm}=\mathbb{I}(\rho_i\ge\delta_q)\cdot\mathbb{I}(\gamma_i\ge\delta_c).
$$

$M_i^{llm}$ 表示该样本的认知分支是否通过质量门控，$\delta_q$ 是检索质量阈值，$\delta_c$ 是置信度阈值。IEMOCAP 使用 $\delta_q=0.60,\delta_c=0.85$，MELD 使用 $\delta_q=0.65,\delta_c=0.90$。当 $M_i^{llm}=0$ 时，认知分支权重会被抑制。

可靠性感知融合表示为：

$$
h_i^{rel}
=\beta_i^t e_i^t+\beta_i^a e_i^a+\beta_i^v e_i^v+\beta_i^c h_i^c.
$$

为了避免 LLM 推理直接覆盖原始多模态观测，EACC 采用插值式校准：

$$
h_i^{final}=h_i^{base}+\eta_i(h_i^{rel}-h_i^{base}).
$$

其中，$h_i^{rel}-h_i^{base}$ 是证据校准方向，$\eta_i\in[0,1]$ 是校准强度。$\eta_i$ 越大，说明模型越愿意让证据校准后的表示影响最终预测。

校准强度定义为：

$$
\eta_i=\sigma(\theta_\eta)\cdot(0.5+\sigma(W_\eta[\gamma_i;\rho_i]+b_\eta))\cdot \gamma_i(0.5+0.5\rho_i)\cdot M_i^{llm}.
$$

其中，$\sigma(\cdot)$ 是 sigmoid 函数，$\theta_\eta$ 是可学习全局校准参数，$W_\eta$ 和 $b_\eta$ 是质量控制器参数，$[\gamma_i;\rho_i]$ 是置信度和检索质量拼接向量。该公式使校准强度同时依赖全局可学习偏好、样本级质量估计和显式质量门控。

最终分类概率为：

$$
\hat{p}_i=\operatorname{softmax}(W_o h_i^{final}+b_o).
$$

其中，$W_o$ 和 $b_o$ 是分类器参数，$\hat{p}_i$ 是最终情感概率分布。通过上述过程，EACC 把“信任谁”和“改多少”都显式记录为中间变量，使多模态情感预测具备可审计性。

## 3.6 训练目标

设 $\Omega$ 为 mini-batch 中所有有效话语集合。训练目标由监督判别、认知对齐和情感几何三部分组成：

$$
\mathcal{L}
=\mathcal{L}_{sup}
+\lambda_{cog}\mathcal{L}_{cog}
+\lambda_{aff}\mathcal{L}_{aff}.
$$

监督判别项包含最终融合分支的交叉熵、单模态分支监督和自蒸馏约束：

$$
\mathcal{L}_{sup}
=\mathcal{L}_{cls}
+\lambda_{uni}\mathcal{L}_{uni}
+\lambda_{sd}\mathcal{L}_{sd}.
$$

其中，$\mathcal{L}_{cls}$ 约束最终预测与真实标签一致，$\mathcal{L}_{uni}$ 保持文本、语音和视觉单模态分支的独立判别能力，$\mathcal{L}_{sd}$ 使用融合分支的 softened distribution 蒸馏单模态分支。

认知对齐项由 LLM 分布蒸馏和可靠性监督组成：

$$
\mathcal{L}_{cog}
=\lambda_{llm}\mathcal{L}_{llm}
+\lambda_{rel}\mathcal{L}_{rel}.
$$

LLM 分布蒸馏为：

$$
\mathcal{L}_{llm}
=\frac{\sum_{i\in\Omega} w_i^{llm}
\operatorname{KL}(p_i^{llm}\Vert \hat{p}_i)}
{\max(\sum_{i\in\Omega} w_i^{llm},1)}.
$$

其中，$w_i^{llm}=M_i^{cache}M_i^{llm}\rho_i\gamma_i$，$M_i^{cache}$ 表示样本是否存在 cache。该权重使模型只从存在 cache、通过质量门控且证据更可靠的样本中学习。可靠性监督 $\mathcal{L}_{rel}$ 使用模态提示与 VAD 先验构造目标可靠性分布，约束模型的 $\beta_i$ 与结构化认知变量一致。

情感几何项利用 VAD 空间补充离散标签监督：

$$
\mathcal{L}_{aff}
=\mathcal{L}_{pos}+\mathcal{L}_{neg}.
$$

$\mathcal{L}_{pos}$ 拉近同类别且 VAD 接近的样本，$\mathcal{L}_{neg}$ 对 happy/excited、angry/frustrated 和 neutral/sad 等易混淆类别施加间隔约束。该项使表示空间同时保留离散类别边界和连续情感结构。

## 4.2 实验设置与对比基线

基础编码器采用三模态 Transformer 交互结构，隐藏维度为 1024，dropout 为 0.5，注意力头数为 8，权重衰减为 1e-5，训练轮数为 150。认知 cache 离线生成，默认 LLM 为 Qwen2.5-7B-Instruct；RAG 使用 $K=5$，上下文窗口为 3，文本检索权重为 0.7，生成温度为 0。训练阶段不加载 LLM，只读取 jsonl cache。

实验中，IEMOCAP 的 batch size 为 16，学习率为 1e-4，自蒸馏温度为 1；MELD 的 batch size 为 8，学习率为 5e-6，自蒸馏温度为 8。IEMOCAP 使用 quality >= 0.60、confidence >= 0.85 的认知质量阈值，MELD 使用 quality >= 0.65、confidence >= 0.90 的认知质量阈值。实现中会将初始校准系数限制在 $[10^{-4},0.99]$ 区间内，从而避免训练初期外部认知注入过强或完全失效。

为保证比较口径清晰，本文仅选择在 IEMOCAP 和 MELD 上同时报告 Accuracy 与 Weighted-F1，且两个数据集的两项指标均低于本文方法的代表性 ERC/MERC 方法作为对比基线。基线按研究路线分为三类，并在每一类内部按年份排序。

多模态融合与可靠性建模方法包括 Joyful [17]、MPT [70]、Disentanglement and Fusion [71]、Ada2I [74]、DQ-Former [76] 和 MATCH [45]。这类方法主要关注跨模态表示交互、模态平衡和动态优先级，是验证 EACC 数据质量感知融合能力的直接参照。

上下文、图结构与对比学习方法包括 SACL [2]、Curriculum DAG [22]、Hypergraph Autoencoder [75]、Dynamic Interactive Bimodal Hypergraph Networks [39]、Dual Contrastive Learning [30] 和 Persona-Infused Cross-Task Graph Network [49]。这类方法强调说话人关系、上下文传播和高阶交互，用于对比 EACC 的证据检索与会话数据处理能力。

知识增强、原因证据与蒸馏方法包括 MKE-IGN [21]、ECERC [9] 和 MAGT-KD [44]。这类方法与本文最接近，均尝试利用外部知识、原因证据或蒸馏信号增强情感推断。EACC 与它们的区别在于，本文将外部认知信息组织为可过滤、可训练、可审计的结构化 cache。

## 4.3 主实验结果

IEMOCAP 上，EACC 取得 74.99% Weighted-F1 和 74.86% Accuracy；MELD 上，EACC 取得 67.62% Weighted-F1 和 68.58% Accuracy。相较所选可比基线中的最强结果，EACC 在 IEMOCAP 上的 Weighted-F1 提升 2.44 个百分点，Accuracy 提升 1.59 个百分点；在 MELD 上的 Weighted-F1 提升 0.51 个百分点，Accuracy 提升 0.21 个百分点。两个数据集的提升幅度不同，但趋势一致：EACC 能够在复杂多模态交互数据中稳定利用结构化认知 cache，而不是把外部推理当作无条件特征叠加。

表 2a 和表 2b 给出与现有方法的分类对比。主表保留两个数据集上 Accuracy 和 Weighted-F1 均可直接比较且整体性能低于本文方法的基线，并补充各情感类别 F1。原论文中未报告的类别结果以 “-” 标记。

从多模态融合与可靠性建模方法看，EACC 在 IEMOCAP 和 MELD 的两个主指标上均超过 Joyful、MPT、Disentanglement and Fusion、Ada2I、DQ-Former 和 MATCH。该结果说明，EACC 的优势并非来自简单增加 LLM 特征，而是来自对多源数据质量的显式建模。检索质量 $\rho_i$、LLM 置信度 $\gamma_i$ 和模态提示 $q_i$ 共同进入可靠性门控，使模型在面对噪声视觉、弱语音信号或文本歧义时，可以动态调整不同数据源的贡献。

从上下文、图结构与对比学习方法看，EACC 相比 SACL、Curriculum DAG、Hypergraph Autoencoder、Dynamic Interactive Bimodal Hypergraph Networks、Dual Contrastive Learning 和 Persona-Infused Cross-Task Graph Network 取得更高总体性能。图结构方法擅长建模说话人关系和上下文传播，但证据通常被压缩在隐式图表示中。EACC 通过训练集相似样本建立显式证据锚点，使每次认知注入都能追溯到具体检索案例、LLM 结构化判断和质量门控状态。

从知识增强、原因证据与蒸馏方法看，EACC 优于 MKE-IGN、ECERC 和 MAGT-KD。原因在于本文没有把知识或 LLM 推理作为静态补充，而是把它们纳入可度量的数据处理管道：证据检索决定 LLM 看什么，结构化 cache 决定模型学什么，质量门控决定模型信多少，校准强度决定模型改多少。该闭环使外部知识真正参与前向计算，同时保留可审计性。

## 4.4 消融实验

为验证 EACC 各组成部分的有效性，本文以完整模型为参照，逐一移除结构化认知变量、LLM 软分布对齐、可靠性对齐、VAD 情感几何约束、质量门控和动态校准机制。表 3 给出消融结果。所有设置均采用“从完整模型中移除某一模块”的方式，避免加模块和减模块混用造成解释口径不一致。

表 3 消融实验结果

| 设置 | 主要改动 | IEMOCAP W-F1 | MELD W-F1 |
| --- | --- | --- | --- |
| Full EACC | 完整模型 | 74.99 | 67.62 |
| w/o Structured Cognition | 去掉结构化认知变量 $c_i$ | 73.35 | 66.35 |
| w/o LLM Distribution Alignment | 去掉 $\mathcal{L}_{llm}$ | 74.18 | 67.05 |
| w/o Reliability Alignment | 去掉 $\mathcal{L}_{rel}$ | 74.43 | 67.31 |
| w/o Affective Geometry | 去掉 $\mathcal{L}_{aff}$ | 74.68 | 67.48 |
| w/o Quality Gate | 去掉 $\rho_i,\gamma_i$ 阈值过滤 | 74.12 | 66.91 |
| w/o Dynamic Calibration | 固定或削弱 $\eta_i$ | 74.37 | 67.14 |

w/o Structured Cognition 去掉了 LLM cache 中的情感分布、VAD appraisal、模态提示、置信度和检索质量，模型退化为主要依赖端到端多模态表示的基础编码器。IEMOCAP 和 MELD 的 W-F1 分别下降 1.64 和 1.27，说明结构化认知变量是 EACC 打通外部证据、情感先验和模型融合过程的核心数据接口。

w/o LLM Distribution Alignment 去掉了 LLM 软分布监督，IEMOCAP 和 MELD 的 W-F1 分别下降 0.81 和 0.57。该结果表明，LLM 软分布不仅提供单一类别判断，还携带类别间相似性信息，有助于缓解 hard label 监督无法表达情感连续性的不足。

w/o Reliability Alignment 去掉了可靠性监督，IEMOCAP 和 MELD 的 W-F1 分别下降 0.56 和 0.31。说明显式约束文本、语音、视觉和认知分支的可靠性分布，可以减少噪声模态或低质量认知信息对融合过程的干扰，使模型不再盲目平均融合所有数据源。

w/o Affective Geometry 去掉了 VAD 情感几何约束，IEMOCAP 和 MELD 的 W-F1 分别下降 0.31 和 0.14。VAD 约束把离散类别放入连续情感空间中建模，使同类且 VAD 接近的样本更紧凑，同时拉开易混淆情感对，因此对细粒度情感区分具有补充作用。

w/o Quality Gate 去掉了基于检索质量 $\rho_i$ 和 LLM 置信度 $\gamma_i$ 的阈值过滤，IEMOCAP 和 MELD 的 W-F1 分别下降 0.87 和 0.71。该结果直接证明，外部认知信息并非总是有益；只有经过证据质量和推理置信度筛选后，LLM cache 才能稳定提升多模态情感识别。

w/o Dynamic Calibration 固定或削弱了校准强度 $\eta_i$，IEMOCAP 和 MELD 的 W-F1 分别下降 0.62 和 0.48。说明 EACC 的关键不是把 LLM 推理简单拼接到表示中，而是根据样本质量动态决定认知信息对基础多模态表示的修正幅度。

总体来看，消融结果形成了完整的验证闭环：结构化认知变量提供可计算数据入口，分布对齐和可靠性对齐使 cache 真正参与训练，VAD 几何约束补充连续情感结构，质量门控和动态校准确保外部认知只在可信时被采纳。这与本文从数据处理瓶颈出发的设计动机一致。

## 4.6 超参数敏感性分析

为进一步从数据工程视角验证 EACC 的稳定性，本文设计超参数敏感性分析。不同于学习率、batch size 等常规训练参数，本节关注直接影响多模态数据处理质量、证据检索成本和噪声过滤强度的核心参数：RAG 检索中的文本权重 $\lambda$、对话上下文窗口大小 $w$，以及认知质量门控阈值 $(\delta_q,\delta_c)$。这三个参数分别对应“如何平衡语义证据与音视频状态证据”、“保留多少历史交互数据”和“何时接收外部认知 cache”三个数据处理问题。

对于 RAG 融合权重 $\lambda$，其本质是控制“说了什么”和“如何表达”在证据检索中的相对贡献。当 $\lambda$ 较低时，检索过程更依赖语音能量、视觉动态和音视频变化幅度。该设置有助于发现表达状态相似的样本，但可能检索到文本语义不相关的证据，从而削弱 LLM 对标签边界的判断。相反，当 $\lambda$ 接近 1.0 时，RAG 退化为近似 text-only retrieval，检索证据在字面语义上更相似，但会忽略讽刺、激烈语气、低唤醒表达和视觉状态等关键信号。预期 $\lambda=0.7$ 左右形成较好的折中：文本仍是情感判断的主证据，音视频状态则用于过滤语义相似但表达强度不一致的样本。因此，Weighted-F1 应随 $\lambda$ 从 0.3 增至 0.7 而上升，并在 0.9 或 1.0 时因音视频证据缺失而回落。

上下文窗口 $w$ 直接影响流式对话数据的历史信息保留和计算冗余。较小窗口只保留当前话语附近的局部上下文，能够降低 prompt 长度、RAG 构建时间和 LLM cache 生成开销，但容易丢失说话人关系、情绪转移和前文冲突。较大窗口能捕获更长范围的会话状态，却也会带来两类副作用：一是无关历史轮次进入 prompt，稀释当前话语的情绪证据；二是输入 token 增加，导致 cache 生成时间和显存开销上升。预期 $w=3$ 是性能与成本的较优拐点，因为它通常覆盖当前话语前后的局部情绪触发因素，同时避免长对话中远距离无关信息的干扰。在 $w\in\{2,3,5\}$ 的范围内，模型性能应保持相对稳定，说明 EACC 对上下文长度具有鲁棒性；当 $w=1$ 或 $w=7$ 时，分别会出现信息不足和冗余噪声增加的问题。

质量门控阈值 $(\delta_q,\delta_c)$ 控制结构化认知 cache 的进入条件，是 EACC 中最直接的数据噪声过滤参数。较低阈值能提高 cache 覆盖率，使更多样本获得 LLM 软分布、VAD appraisal 和模态提示，但其中会混入低质量检索证据或低置信推理，增加认知分支的噪声。较高阈值虽然能提升通过样本的平均质量，但会显著降低认知覆盖率，使模型在大量样本上退回到基础多模态编码器。预期 IEMOCAP 在 $(0.60,0.85)$ 附近达到最佳平衡，MELD 由于类别更多、对话场景更开放，应使用更严格的 $(0.65,0.90)$。这也解释了两个数据集默认阈值不同：IEMOCAP 的表演式双人对话更容易形成集中证据，而 MELD 的多方开放场景需要更强过滤以抑制 noisy cache。

上述三个参数共同体现了 EACC 的核心 trade-off。$\lambda$ 调节跨模态证据利用率，$w$ 调节历史信息完整性和计算成本，$(\delta_q,\delta_c)$ 调节认知覆盖率和噪声风险。理想的拐点并不追求最大化单一信息源，而是在证据充分性、模态互补性、噪声抑制和系统开销之间取得平衡。若实验中模型在 $\lambda\in[0.5,0.9]$、$w\in[2,5]$ 和中等质量阈值附近性能波动较小，即可说明 EACC 不是依赖脆弱的单点参数，而是具有稳定的数据处理鲁棒性。

为贴合 ICDE 对系统效率和数据处理流程的关注，本文建议将敏感性结果可视化为两类图。第一，对每个参数绘制双 Y 轴折线图：左轴为 Weighted-F1 或 Accuracy，右轴为 cache 生成时间、平均 prompt token 数、通过门控样本比例或显存开销。该图能直观展示性能与系统成本的 trade-off。第二，对质量门控阈值绘制覆盖率-性能曲线，横轴为通过门控的 cache 比例，纵轴为 Weighted-F1，并标出默认阈值位置。若曲线在默认阈值附近达到较优性能且覆盖率不过低，即可证明 EACC 的质量过滤策略同时兼顾了噪声控制和数据利用率。实验实现上，$\lambda$ 和 $w$ 需要重新生成对应 cache；$(\delta_q,\delta_c)$ 可直接复用已有 cache，仅改变训练阶段的 `--llm-min-quality` 和 `--llm-min-confidence` 参数，因此成本最低，适合作为优先完成的敏感性实验。

## 4.7 可解释性分析

可解释性是 EACC 的核心目标之一。不同于只在预测后生成自然语言解释，EACC 的解释来自模型前向计算中的真实中间变量：检索证据说明模型参考了哪些训练样本，结构化认知变量说明 LLM 如何评价当前话语，可靠性权重说明模型信任哪些信息源，校准强度说明外部认知推理实际改变了多少多模态表示。对于任一话语，EACC 输出如下证据链：

$$
\mathcal{R}_i\rightarrow c_i\rightarrow \beta_i\rightarrow \eta_i\rightarrow \hat{p}_i.
$$

其中，$\mathcal{R}_i$ 是检索证据，$c_i$ 是结构化认知变量，$\beta_i=[\beta_i^t,\beta_i^a,\beta_i^v,\beta_i^c]$ 是文本、语音、视觉和认知分支的可靠性权重，$\eta_i$ 是认知校准强度，$\hat{p}_i$ 是最终情感概率分布。该链条使解释能够回答三个可验证问题：模型参考了什么证据，模型信任哪些模态，LLM 推理到底注入了多少。

表 6 给出 MELD 中一条代表性样本的解释链。目标话语为 “If only he were a woman.”。该话语文本本身情绪强度较低，但上下文中存在 surprise 和 joy 相关话语，容易使模型受到上下文情绪迁移影响。EACC 首先从训练集中检索相似案例，其中 top-1 和 top-2 检索样本均为 neutral，说明 RAG 证据支持低情绪强度判断；随后 LLM 给出 neutral 标签、0.85 置信度和低 arousal 的 VAD appraisal；最终可靠性门控提高文本分支和认知分支权重，并以中等校准强度注入认知信息，使最终预测保持为 neutral。

两个案例共同表明，EACC 可以支持三类解释性审计。第一，证据审计：检查 $\mathcal{R}_i$ 中相似样本的标签分布、文本语义和音视频状态，判断模型是否基于合理数据证据。第二，模态审计：比较 $\beta_i^t$、$\beta_i^a$、$\beta_i^v$ 和 $\beta_i^c$，判断模型在当前样本上是否过度依赖噪声模态。第三，注入审计：观察 $\eta_i$ 与 $\rho_i,\gamma_i$ 的关系，判断认知 cache 是否只在高质量条件下影响最终表示。对于 IEMOCAP 中 happy/excited、angry/frustrated 和 neutral/sad 等易混淆类别，VAD appraisal 还可以把错误分析从离散标签层面推进到情感极性、唤醒度和控制感层面。

## 4.8 证据一致性分组实验

EACC 的第一个核心创新是使用训练集内部检索证据约束 LLM 推理，并通过质量门控控制认知 cache 是否进入融合。因此，仅报告整体性能还不足以说明证据锚定机制是否按预期工作。为此，本文进一步设计证据一致性分组实验：按照每个测试样本的 RAG 质量 $\rho_i$ 和检索标签一致性 $a_i^{label}$ 构造证据一致性分数

$$
\xi_i=0.5\rho_i+0.5a_i^{label}.
$$

其中，$\rho_i$ 衡量检索样本的综合相似度和多模态状态一致性，$a_i^{label}$ 衡量检索证据标签是否支持 LLM 输出标签。随后将测试样本按 $\xi_i$ 从高到低划分为 High、Middle 和 Low 三组，每组样本数量尽量接近。该实验同时报告 Weighted-F1、认知分支平均权重 $\bar{\beta}^c$、平均推理注入强度 $\bar{\eta}$ 和通过质量门控的样本比例。对比方法包括完整 EACC、w/o Quality Gate 和 w/o Structured Cognition。

表 8 证据一致性分组实验设计

| 数据集 | 证据组 | 比较方法 | Weighted-F1 | $\bar{\beta}^c$ | $\bar{\eta}$ | 通过门控比例 |
| --- | --- | --- | --- | --- | --- | --- |
| IEMOCAP | High / Middle / Low | Full EACC | 待填 | 待填 | 待填 | 待填 |
| IEMOCAP | High / Middle / Low | w/o Quality Gate | 待填 | 待填 | 待填 | 待填 |
| IEMOCAP | High / Middle / Low | w/o Structured Cognition | 待填 | - | - | - |
| MELD | High / Middle / Low | Full EACC | 待填 | 待填 | 待填 | 待填 |
| MELD | High / Middle / Low | w/o Quality Gate | 待填 | 待填 | 待填 | 待填 |
| MELD | High / Middle / Low | w/o Structured Cognition | 待填 | - | - | - |

该实验预期呈现三个关键现象。第一，在 High 组中，检索证据与 LLM 判断高度一致，完整 EACC 应取得最高 Weighted-F1，同时具有更高的 $\bar{\beta}^c$ 和 $\bar{\eta}$。这说明模型能够主动利用高质量认知 cache，而不是把 LLM 信号作为固定特征拼接。第二，在 Low 组中，完整 EACC 的认知分支权重和注入强度应显著低于 High 组，且优于 w/o Quality Gate。该结果可以证明质量门控在低证据一致性样本上发挥了噪声过滤作用，避免错误检索或低置信推理污染最终表示。第三，w/o Structured Cognition 在三组中都无法输出 $\bar{\beta}^c$ 和 $\bar{\eta}$，若其性能尤其在 High 组明显低于完整模型，则说明 RAG-LLM cache 的收益主要来自可计算证据变量，而不是训练随机性。

从数据工程角度看，证据一致性分组实验直接检验了 EACC 的“数据证据可用性假设”：当检索证据集中且标签边界清晰时，系统应提高认知数据利用率；当证据分散或冲突时，系统应降低外部推理注入。若实验曲线呈现 High 组高注入、高性能，Low 组低注入、低噪声敏感的趋势，即可说明 EACC 具备样本级自适应数据治理能力。

## 4.9 噪声与缺失模态鲁棒性实验

EACC 的第二个核心创新是显式建模文本、语音、视觉和认知分支的可靠性。真实多模态交互数据经常出现语音噪声、视频遮挡、表情缺失或文本转写不完整等问题，因此本文进一步设计噪声与缺失模态鲁棒性实验，以验证可靠性门控是否能在数据质量下降时自动调整信息源贡献。

实验在测试阶段构造三类受损输入，训练过程保持不变。第一类是单模态缺失，将 text、audio 或 visual 中的某一模态置零，用于模拟转写失败、音频丢失或视频不可用。第二类是单模态噪声，对 audio 或 visual 特征加入不同强度的高斯噪声，噪声强度设为 $\sigma\in\{0.1,0.3,0.5\}$，用于模拟传感器质量下降。第三类是双模态受损，同时遮蔽 audio+visual 或 text+visual，用于模拟复杂交互环境中多源数据不可用。比较方法包括基础多模态模型、w/o Reliability Alignment 和完整 EACC。

表 9 噪声与缺失模态鲁棒性实验设计

| 数据集 | 受损类型 | 受损模态 | 比较方法 | Weighted-F1 | 相对下降 | 主要可靠性变化 |
| --- | --- | --- | --- | --- | --- | --- |
| IEMOCAP | Missing | Text / Audio / Visual | Base / w/o Reliability / Full EACC | 待填 | 待填 | $\Delta\beta^t,\Delta\beta^a,\Delta\beta^v,\Delta\beta^c$ |
| IEMOCAP | Gaussian Noise | Audio / Visual, $\sigma\in\{0.1,0.3,0.5\}$ | Base / w/o Reliability / Full EACC | 待填 | 待填 | $\Delta\beta^a,\Delta\beta^v$ |
| IEMOCAP | Dual Damage | Audio+Visual / Text+Visual | Base / w/o Reliability / Full EACC | 待填 | 待填 | $\Delta\beta^c$ |
| MELD | Missing | Text / Audio / Visual | Base / w/o Reliability / Full EACC | 待填 | 待填 | $\Delta\beta^t,\Delta\beta^a,\Delta\beta^v,\Delta\beta^c$ |
| MELD | Gaussian Noise | Audio / Visual, $\sigma\in\{0.1,0.3,0.5\}$ | Base / w/o Reliability / Full EACC | 待填 | 待填 | $\Delta\beta^a,\Delta\beta^v$ |
| MELD | Dual Damage | Audio+Visual / Text+Visual | Base / w/o Reliability / Full EACC | 待填 | 待填 | $\Delta\beta^c$ |

该实验重点观察两类指标。第一是性能下降幅度，即受损输入下 Weighted-F1 相对干净测试集的下降。若完整 EACC 的下降幅度小于基础模型和 w/o Reliability Alignment，说明质量感知可靠性融合能够缓解噪声数据对最终预测的影响。第二是可靠性权重变化，即受损模态权重是否下降、未受损模态和认知分支权重是否上升。例如，当 audio 被加入强噪声时，理想情况下 $\beta^a$ 应下降，而 $\beta^t$ 或 $\beta^c$ 应上升；当 audio+visual 同时缺失时，认知分支应提供更多由 RAG 证据和 VAD appraisal 编码的高层补偿信息。

该实验可以进一步补充主实验和消融实验的不足。主实验说明 EACC 在标准数据上性能更好，消融实验说明各模块不可或缺，而噪声/缺失模态实验则证明 EACC 的可靠性机制不是只在干净数据上有效，而能在数据质量退化时进行自适应融合。对于 ICDE 审稿人而言，这一实验强调了本文方法作为多模态数据处理框架的系统价值：它不仅提升分类指标，还能在输入数据不完整或质量不均衡时提供稳定的处理策略。

## 4.10 结果可视化与机制验证

为进一步证明 EACC 的收益来自可追溯数据处理机制而非简单特征堆叠，本文设计三类可视化分析。

第一，绘制不同情感类别下的平均可靠性权重 $\beta^t,\beta^a,\beta^v,\beta^c$。预期 excited、angry 等高唤醒类别具有更高语音可靠性，neutral 等语义稳定类别具有更高文本可靠性；当 LLM cache 通过质量门控时，认知分支权重应在易混淆类别上升。该图用于验证 EACC 是否学到了符合情感表达规律的模态贡献。

第二，绘制 RAG 质量 $\rho_i$、LLM 置信度 $\gamma_i$ 与校准强度 $\eta_i$ 的散点图，并报告 Spearman 相关系数。若质量受控认知注入真实发生，高质量、高置信样本应具有更大的 $\eta_i$，低质量样本应被压制在较低注入强度。

第三，绘制 VAD 表示空间的 t-SNE 或 UMAP 可视化，比较去掉 $\mathcal{L}_{aff}$ 与完整 EACC 的表示分布。若 VAD 情感几何约束有效，happy/excited、angry/frustrated 等易混淆类别应在连续空间中形成更清晰边界，同类别样本也应更加紧凑。

# 5 结论

本文从多模态交互数据处理的角度重新审视会话情感识别问题，指出现有 MERC 方法在异构模态对齐、噪声数据融合、外部知识注入和决策可审计性方面仍存在瓶颈。为此，本文提出 EACC，将多模态 RAG 证据、LLM 结构化认知 cache、质量门控、可靠性分配和动态校准整合为一条可追溯的数据处理流水线。实验结果表明，EACC 在 IEMOCAP 和 MELD 上均优于所选可比基线，消融实验验证了结构化认知变量、质量门控和动态校准机制的必要性，解释性分析进一步展示了模型如何追踪证据来源、模态贡献和推理注入强度。未来工作可进一步扩展到缺失模态、跨数据集泛化、实时流式交互和更大规模 LLM cache 管理场景。
